# クロスデバイス対応リーディングリスト管理ツール: システム要求仕様書

## ステータス

Confirmed

## 日付

2026-02-19

## 1. システム境界

### 1.1 システム概要

記事URLの保存・未読/既読管理・検索・重複検出・未読蓄積警告をスマホとPC間でシームレスに提供するクロスデバイス対応リーディングリスト管理ツール。

### 1.2 アクター一覧

| ID | アクター | 種別 | 概要 |
|----|---------|------|------|
| ACT-1 | ユーザー | 人間 | 記事の保存・管理・閲覧を行う唯一の利用者 |

### 1.3 外部インターフェース

| ID | インターフェース | 接続先 | 方向 | 概要 |
|----|-----------------|--------|------|------|
| IF-1 | スマホ操作 | ACT-1 | 双方向 | スマホからの記事保存・未読管理・検索操作 |
| IF-2 | PC操作 | ACT-1 | 双方向 | PCからの記事保存・未読管理・検索操作 |
| IF-3 | 記事Webサイト | 外部Webサイト | 入力 | 保存URLから記事タイトル等のメタ情報を取得 |

### 1.4 スコープ

**システム名**: クロスデバイス対応リーディングリスト管理ツール

**内部（システムが担う範囲）:**
- 記事URLの保存とメタ情報の取得・管理
- 未読/既読状態の管理
- 記事の検索
- 重複URLの検出と警告
- 未読件数の表示と蓄積警告
- スマホ・PC間のデータ同期

**制約:** ユーザー要求書（`docs/project-definition/problem-statement.md`）の制約セクションを参照

**外部（システムが担わない範囲）:**
- 記事の発見（RSSリーダー・X・Web閲覧はユーザーが行う）
- AI追加機能（おすすめ度評価、要約、タグ付け等 — スコープ外）
- リンク切れの自動検出
- 別URLの同一記事検出

## 2. 機能要求

| SR-ID | 要求 | EARS型 | トレース元 |
|-------|------|--------|-----------|
| SR-001 | ユーザーがURLを入力して保存を実行したとき、システムは対象URLから記事タイトルおよびメタ情報を取得して保存すること | Event-Driven | UC-1 |
| SR-002 | ユーザーがURLを入力して保存を実行したとき、システムは保存日時を記録すること | Event-Driven | UC-1 |
| SR-003 | ユーザーが未読一覧の表示を要求したとき、システムは保存済みの未読記事のタイトル・URLおよび保存日時を一覧表示すること | Event-Driven | UC-2 |
| SR-004 | 未読記事が0件の間、システムは未読記事が存在しない旨を一覧画面に表示すること | State-Driven | UC-2 |
| SR-005 | ユーザーが記事を既読にする操作を実行したとき、システムは対象記事の状態を既読に更新し、未読一覧から除外すること | Event-Driven | UC-3 |
| SR-006 | ユーザーが既読記事を未読に戻す操作を実行したとき、システムは対象記事の状態を未読に更新し、未読一覧に復帰させること | Event-Driven | UC-4 |
| SR-007 | ユーザーが記事の削除を実行したとき、システムは対象記事を一覧から削除すること | Event-Driven | UC-5 |
| SR-008 | ユーザーがキーワードを入力して検索を実行したとき、システムは保存済み記事のタイトルおよびURL内にそのキーワードを含む記事を抽出して一覧表示すること | Event-Driven | UC-6 |
| SR-009 | 検索結果が0件の場合、システムは該当記事が存在しない旨を表示すること | Unwanted Behaviour | UC-6 |
| SR-010 | ユーザーが既に保存済みのURLを再度保存しようとしたとき、システムは重複を検出し警告メッセージを表示して保存を中止すること | Unwanted Behaviour | UC-7 |
| SR-011 | システムは未読記事の件数をユーザーが確認できる画面に常に表示すること | Ubiquitous | UC-8 |
| SR-012 | 未読記事件数が50件を超えたとき、システムは未読が蓄積している旨の警告を表示すること | Event-Driven | UC-9 |
| SR-013 | 入力されたURLの形式が不正な場合、システムはエラーメッセージを表示して保存を中止すること | Unwanted Behaviour | UC-10 |
| SR-014 | 記事メタ情報の取得に失敗した場合、システムはエラーメッセージを表示してURLのみ保存する、またはエラーとして保存を中止するかをユーザーに選択させること | Unwanted Behaviour | UC-10 |
| SR-015 | 保存操作中にネットワーク接続エラーが発生した場合、システムはエラーメッセージを表示して保存を中止すること | Unwanted Behaviour | UC-10 |

## 3. 非機能要求

| NFR-ID | 観点 | 要求 | 測定基準 | 測定方法 | 導出根拠 |
|--------|------|------|---------|---------|---------|
| NFR-001 | コスト | システムの運用にかかる月額費用は0円であること（Claude API利用料は既存契約に含まれるため除外） | 月額0円 | 月次の利用料金明細で確認 | 個人利用の制約。既存Claude API契約は利用可能 |
| NFR-002 | 運用負荷 | リリース作業は手動操作3ステップ以内で完了できること | 手動操作3ステップ以内 | リリース手順書のステップ数で確認 | 個人運用で運用者=開発者。頻繁なリリースを想定し手間を最小化 |
| NFR-003 | パフォーマンス | 主要操作（保存・一覧表示・検索）の応答時間は3秒以内であること | 応答時間3秒以内 | ブラウザのネットワークタブまたはログで応答時間を計測 | Nielsenの応答時間閾値を参考。個人利用で「遅すぎない」は3秒程度が妥当 |
| NFR-004 | セキュリティ | 認証情報（APIキー等）はシステムの公開領域に平文で保存しないこと | 認証情報が公開領域に平文で存在しない | ソースコードレビューおよびデプロイ構成の確認 | 個人利用で最低限のセキュリティ。認証情報漏洩防止が最低要件 |
| NFR-005 | 可用性 | 計画外停止の復旧時間（RTO）は24時間以内。データ損失の許容範囲（RPO）は直近のバックアップ時点まで | RTO 24時間以内、RPO 直近バックアップ | 障害発生時の復旧時間の記録 | 個人利用で停止許容。データ消失は回避 |
| NFR-006 | ユーザビリティ | 記事の保存操作はURL入力から完了まで3ステップ以内で行えること | 操作ステップ3以内 | 操作フロー上のステップ数で確認 | UC-1「素早く保存する」の要求を定量化 |
| NFR-007 | データ整合性 | あるデバイスで行った操作が、他のデバイスから次回アクセス時に反映されていること | 次回アクセス時に最新データが反映 | デバイスAで操作後、デバイスBで最新データを確認 | クロスデバイス対応の中核要件 |

## 付録

### A. 用語集

| 用語 | 定義 |
|------|------|
| SR | System Requirement（システム要求）。システムが満たすべき機能要求の識別子 |
| NFR | Non-Functional Requirement（非機能要求）。性能・セキュリティ・可用性等の品質要求 |
| IR | Interface Requirement（インターフェース要求）。外部インターフェースに関する要求 |
| EARS | Easy Approach to Requirements Syntax。要求記述の標準的な構文パターン（Event-Driven / State-Driven / Unwanted Behaviour / Ubiquitous 等） |
| UC | Use Case。ユーザー要求書で定義されたユースケース識別子 |
| ACT | Actor（アクター）。システムと相互作用する人間または外部システムの識別子 |
| IF | Interface（インターフェース）。システムと外部要素との接続点の識別子 |
| 未読 | ユーザーが保存した記事のうち、まだ既読操作を行っていない状態 |
| 既読 | ユーザーが既読操作を実行した記事の状態 |
| メタ情報 | 記事URLから取得する付加情報。主に記事タイトルを指す |
| クロスデバイス | スマホとPCなど、複数の異なる端末間でデータを共有・同期する仕組み |
| RTO | Recovery Time Objective（目標復旧時間）。障害発生からシステム復旧までの目標時間 |
| RPO | Recovery Point Objective（目標復旧時点）。障害発生時に許容するデータ損失の範囲（時点） |
