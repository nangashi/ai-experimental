# 決定ステートメント

## 議題
シングルユーザーWebアプリの認証・アクセス制御として、どの認証方式を採用するか

## スコープ
- 対象: 認証方式（Cloudflare Access / 自前JWT / Basic認証等）の選定、セッション管理方式、Web Share Target APIからのPOSTリクエストとの互換性
- 対象外: 認可（権限管理）の詳細設計（シングルユーザーのため不要）、アプリケーション層のビジネスロジック

## 可逆性
medium
認証方式はミドルウェアレベルで全エンドポイントに影響するが、シングルユーザーのためユーザーデータのマイグレーションは不要。Hono のミドルウェア差し替えで認証方式を変更可能。
