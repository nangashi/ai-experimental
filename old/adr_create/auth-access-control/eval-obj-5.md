# OBJ-5: コスト持続可能性 — 評価結果

## 提案した代替案

- **Hono + Cookie セッション（Workers KV なし・メモリレスステートレス実装）**: パスワードハッシュを Workers Secrets に保存し、セッション情報を HttpOnly Cookie（署名付き JWT 形式）に持たせることで Workers KV を不要にする。外部サービス・追加ストレージへの依存をゼロにすることでランニングコストを完全に固定（Workers の無料枠のみ）できる。

- **Cloudflare Access（Zero Trust 無料プラン）**: Cloudflare が提供する外部認証サービスを利用。シングルユーザーのため無料枠（50 ユーザー上限）内に収まり、追加の Workers KV や D1 への認証データ書き込みも不要。ただし将来のサービス条件変更リスク（S5）を持つ。

- **Basic 認証（Hono 組み込み + HTTPS）**: Hono の組み込み機能または独自実装で Basic 認証を実現。コード量が最小で追加インフラが不要。セッション管理コスト（KV 読み書き）も発生しない。コスト観点での余分な依存が最も少ない選択肢。

## 評価

### Hono + Cookie セッション（Workers KV なし・署名付き Cookie）

- 評価: ◎
- 利点:
  - Workers Secrets（無料）のみを使用し、追加の有料サービスに依存しない。Workers 無料枠（100,000 リクエスト/日）の範囲内で認証を完結できる。
  - セッション状態を Cookie（署名付き JWT）に持たせることで Workers KV の読み書きが不要になる。KV の無料枠（読み取り 100,000/日、書き込み 1,000/日）を消費しないため、アプリの他の用途（ブックマーク読み書き等）に無料枠をフル活用できる。
  - 将来の料金プラン変更リスクがない。Cloudflare Workers の無料プランが継続する限り（これはアプリ全体のホスティング前提でもある）追加費用は発生しない。
- 欠点:
  - トークン失効（ログアウト・強制無効化）を実現するには失効リストの保存先が必要になり、KV や D1 に依存が生じうる。ステートレス Cookie のみで完結させる場合、即時ログアウトが困難（有効期限切れまでトークンが有効）。
  - Cookie の有効期限設計を適切に行わないと、セキュリティ上の問題が生じうる（これはコスト観点の欠点ではなく他の目的との関連）。
- 根拠: Workers 無料枠は認証用途の追加コストが実質ゼロであり、サードパーティ依存がないため料金変更リスクも最低。コスト持続可能性の観点で最も堅牢な選択肢。
- CSD依存:
  - S3（Workers Secrets の安全性）: Secrets が廃止・有料化されるリスクは現実的には低いが理論上は存在する。ただしこれは Workers 自体の廃止と同義であり、アプリ全体の前提崩壊に相当するため、このリスクは無視可能と判断する。
  - S6（Workers KV の無料枠）: 本案では KV を使用しないため、KV の無料枠変更はこの案に影響しない。

### Cloudflare Access（Zero Trust 無料プラン）

- 評価: ○
- 利点:
  - 現時点ではシングルユーザー構成のため、無料枠（50 ユーザー）内に確実に収まる（research.md 観点1）。
  - 追加の Workers KV・D1 書き込みが認証のためには不要。Cloudflare の既存インフラ内で完結するため、Cloudflare 全体が有料化しない限りコスト増はない。
  - 認証処理自体のコスト（Workers の CPU 時間）が削減される。Cloudflare のエッジで認証が完結するため Workers 側のリクエスト処理が軽減される。
- 欠点:
  - **S5 リスクが最大の懸念**: 無料枠の条件変更（ユーザー数上限の引き下げ・機能の有料化）が発生した場合、即座に課金が発生する可能性がある。51 ユーザー目で全席分 $7/席/月（51 席で $357/月）という急激な課金構造（research.md 観点1）は、将来のマルチユーザー化（S1）と組み合わさると致命的なコスト増になりうる。
  - サービス条件が Cloudflare 側の一方的な変更に依存しており、コントロールがない。
  - 現時点では無料だが、Zero Trust サービスはエンタープライズ向けにシフトしており、個人向け無料枠の長期継続性には不確実性がある（S5）。
- 根拠: 現時点のコストはゼロだが、外部サービス依存による将来の料金変更リスク（S5）が他の自前実装案より高い。シングルユーザーが維持される限り問題ないが、S5 の不確実性を考慮して ○ とする。
- CSD依存:
  - S5（Cloudflare Access の料金プラン継続）に強く依存。このSuppositionが崩れた場合、評価は × に下がる。
  - S1（シングルユーザー維持）: マルチユーザー化した場合のコスト増が急激（段階課金でなく全席分）であるため、S1 の崩壊と S5 の崩壊が重なると特に深刻。

### Basic 認証（Hono + HTTPS）

- 評価: ◎
- 利点:
  - 追加の外部サービス・ストレージが一切不要。Workers Secrets にパスワードを保存するだけで完結し、追加コストが発生しない。
  - 最もシンプルな実装のため、Workers の CPU 時間消費も最小。
  - 業界標準の組み込み認証方式であり、廃止・有料化のリスクがない。コスト持続可能性の観点で最も安定している。
  - セッション管理が不要（ステートレス）なため、KV 等のコストが発生しない。
- 欠点:
  - Web Share Target API からの POST リクエストへの対応（OBJ-1 の観点）が困難であり、コスト以外の目的でのトレードオフが大きい。これはコスト観点の欠点ではないが、総合評価での選択に影響する。
  - ブラウザが Basic 認証の資格情報を保存する際、アプリ独自のログイン画面を提供できない（UX 上の制約）。これもコスト観点の問題ではない。
- 根拠: コスト持続可能性の観点では、依存するものが HTTP 仕様そのものだけであり最も安定。ただし他の目的（特に OBJ-1 の Web Share Target API 互換性）での制約が大きく、実際の選択は他の目的とのバランスで判断する必要がある。
- CSD依存:
  - なし（コスト観点での CSD 依存はない。C9 の HTTPS 保証はすべての案の共通前提）。

### 参考: Hono + Cookie セッション（Workers KV を使う実装）

※ 他の Objective Evaluator が提案する可能性のある代替案として、KV を使ったセッション管理の評価も記録する。

- 評価: ○
- 利点:
  - Workers KV は無料枠（読み取り 100,000/日・書き込み 1,000/日）の範囲内ではコストがかからない（S6）。
  - セッション失効（ログアウト・強制無効化）が実現でき、セキュリティ機能が充実する。
- 欠点:
  - **S6 の依存**: KV の無料枠が変更された場合、認証セッションの読み書きコストが発生しうる。セッション発行・検証の頻度次第では無料枠を超える可能性がある（シングルユーザー・個人用途では実際には超えにくいが）。
  - KV の読み書きが Workers のリクエスト処理時間に加算される（OBJ-4 レイテンシへの影響も生じる）。
- 根拠: コスト観点では KV 無料枠内に収まる可能性が高く実用上は問題ないが、KV を不要にする実装（署名付き Cookie）と比べると S6 への依存が生じる分、コスト持続可能性で若干劣る。
- CSD依存:
  - S6（Workers KV の無料枠継続）に依存。

| 代替案 | OBJ-5 評価 | 主な根拠 |
|--------|-----------|---------|
| Hono + Cookie セッション（KV なし） | ◎ | 追加サービス依存ゼロ、料金変更リスクなし |
