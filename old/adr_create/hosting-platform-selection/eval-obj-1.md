# OBJ-1: コスト制約の遵守 — 評価結果

## 提案した代替案

OBJ-1（月額0円での継続運用）を最優先する観点から、以下の代替案を提案する。

- **ALT-A: Cloudflare Pages（FE） + Cloudflare Workers（BE）**: フロントエンドとバックエンドをともにCloudflareのサービスで統一する。Cloudflare Pagesは静的サイト/SPAを永続的な無料枠で提供。Cloudflare WorkersはエッジコンピューティングでのREST API実行を月10万リクエストまで無料で提供する。コールドスタートが事実上なく（エッジランタイム）、月額0円の目的に最も直接的に応える。

- **ALT-B: Netlify（FE） + Render.com（BE、無料枠）**: フロントエンドにNetlify無料プラン、バックエンドにRender.com無料枠を組み合わせる。両者とも永続的な無料枠を持つ。ただしRender.comのスリープ仕様（C7/S5）によりコールドスタート問題が発生する。月額0円を達成できるが、C5（3秒以内の応答）との両立は要検証。

- **ALT-C: Vercel（FE） + Render.com（BE、無料枠）**: フロントエンドにVercel Hobbyプラン（非商用限定の永続無料）、バックエンドにRender.com無料枠を組み合わせる。広く採用されている構成だが、D1（商用利用の有無）が判明した場合にVercelの利用可否が変わるリスクがある（C6）。

- **ALT-D: Cloudflare Pages（FE） + Render.com（BE、有料 $7/月）**: フロントエンドは無料、バックエンドは常時稼働のために有料プランを使う構成。OBJ-1（月額0円）を満たさないため代替案として提示するが、OBJ-2（レスポンス速度）とのトレードオフを示す目的で参照する。

## 評価

### ALT-A: Cloudflare Pages（FE） + Cloudflare Workers（BE）

- 評価: ◎
- 利点:
  - Cloudflare PagesもCloudflare Workersも、永続的かつ無制限の無料枠を提供している（Workersは月10万リクエスト/日を無料枠内で提供）。
  - 期間限定クレジットではなく、継続的な無料利用が可能。
  - Cloudflareは業界最大規模のCDN・インフラ企業であり、サービス継続性のリスクが低い（OBJ-4にも貢献）。
  - フロントエンド・バックエンドを同一エコシステムで管理でき、商用利用制限が他社より緩い（Workersの場合、商用利用も基本的に許容されている）。
- 欠点:
  - Cloudflare WorkersはV8 Isolateベースのエッジランタイムであり、Node.js互換ではあるが一部APIに制限がある（OBJ-5のベンダーロックインリスクに関係）。
  - Workersの無料枠はリクエスト数上限があるため、利用量が増加した場合にコストが発生する可能性がある（月10万リクエスト/日を超える場合）。
  - REST APIが複雑な場合、Workersの実行時間制限（無料で10ms CPU時間）が制約になりうる。
- 根拠: research.mdには直接的なCloudflare Workers価格情報が含まれていないため、一般的な知識を補足として参照している。Cloudflare公式情報では、Workersの無料枠は月10万リクエスト/日まで、Pages は静的サイトで無制限ビルド。
- CSD依存: S3（料金体系の安定性への仮定）に依存する。Cloudflareが無料枠を縮小した場合、この評価は変化する。D1（商用利用の有無）が商用と判明した場合、Cloudflareは商用利用を制限していないため、他のALTと比較してリスクが低い。

---

### ALT-B: Netlify（FE） + Render.com（BE、無料枠）

- 評価: ○
- 利点:
  - Netlifyの無料プランは永続的に利用可能（月300ビルドクレジット、カスタムドメイン、SSL含む）。
  - Render.comの無料枠も永続的に利用可能であり、月額0円を達成できる。
  - D1（商用利用の有無）について、NetlifyはVercelと比較して商用利用制限が緩い（ただしD2として要確認）。
- 欠点:
  - Render.comのバックエンド無料枠はC7（スリープ仕様）・S5（コールドスタート約25秒）の影響を受ける。C5（3秒以内の応答）を満たすためには、Uptime Robot等による定期pingが必要となるが、これは追加の設定負荷を意味する（OBJ-3に影響）。
  - 定期pingによるスリープ回避が規約上許容されるかはRender.comの利用規約次第（リスクあり）。
- 根拠: research.md（観点1、観点3）に基づく。Netlify無料プランの月300ビルドクレジット制限は、小規模個人プロジェクトでは通常問題にならない。
- CSD依存: C7（Render.comスリープ仕様）・S5（コールドスタート25秒）・S3（料金体系の安定性）に依存。D2（Netlify/Render商用利用条件）が商用であることを制限する場合、評価はD1次第で下がる。

---

### ALT-C: Vercel（FE） + Render.com（BE、無料枠）

- 評価: △
- 利点:
  - Vercel Hobbyプランは個人・非商用利用であれば永続的な無料枠として利用可能（C6）。
  - フロントエンドの配信性能・CDN展開においてVercelは高い評価を持つ。
  - Render.com無料枠と組み合わせることで月額0円を維持できる（D1が非商用の場合）。
- 欠点:
  - C6により、Vercel HobbyはD1（商用利用の有無）に強く依存する。商用と判明した場合、Vercel Proは$20/月/ユーザーとなり、C3（月額0円）の目的に完全に反する。
  - Render.comのコールドスタート問題はALT-Bと同様（C7/S5）。
  - OBJ-1の達成確実性がALT-AやALT-Bより低い（D1の不確実性がOBJ-1の評価を条件付きにする）。
- 根拠: research.md（観点1、観点3、観点4）に基づく。Vercel Hobbyの非商用限定規約は公式ドキュメントで確認済み（C6）。
- CSD依存: D1（商用利用の有無）に強く依存。D1が商用と判明した場合、この選択肢はOBJ-1を満たせなくなる（C3との矛盾）。

---

### ALT-D: Cloudflare Pages（FE） + Render.com（BE、有料 $7/月）

- 評価: ×
- 利点:
  - バックエンドが常時稼働になるため、C5（3秒以内の応答）を確実に満たすことができる。
  - OBJ-2（レスポンス速度）の観点では優れた選択肢。
- 欠点:
  - C3（月額0円）を満たさない。バックエンドのRender有料プランは最低$7/月の固定費用が発生する。
  - OBJ-1の目的を直接的に達成できない。
- 根拠: research.md（観点3）に基づく。Render有料プランは$7/月〜で常時稼働が保証される。
- CSD依存: なし（月額コストが発生することは確実であり、C3との矛盾は不確実性に依存しない）。
