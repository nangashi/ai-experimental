# 関連する既存決定

## 関連ADR

### ADR-0001: PWA+REST API構成のホスティング基盤選定
- 決定: Cloudflare Pages（フロントエンド）+ Cloudflare Workers（バックエンド）を採用する。バックエンドは Cloudflare Workers 上で Hono + TypeScript による REST API として動作する。
- 今回の議題との関連: バックエンドが Cloudflare Workers（V8 Isolate ランタイム）上で動作するため、採用するデータベースは Workers ランタイムと互換性を持つ必要がある。Workers は通常の Node.js ランタイムではなく、TCP ソケットの直接使用に制限があるため、従来型の TCP 接続ベースのデータベースドライバがそのままでは動作しない。
- 制約への影響（Certainty として扱うべき事項）:
  - データベースは Cloudflare Workers（V8 Isolate ランタイム）から接続可能な方式でなければならない。具体的には、HTTP/HTTPS ベースの API を持つデータベースサービス、または Cloudflare が公式に提供する Bindings（D1、KV、R2 等）に限定される
  - 月額コストは 0 円（無料ティア内）の制約が引き続き適用される（ADR-0001 の C3 より）
  - シングルユーザー構成であり、大規模スケーリングは不要（ADR-0001 の C4 より）
  - 画面表示 3 秒以内の要件が引き続き適用されるため、データベースのレイテンシもこの制約に含まれる（ADR-0001 の C5 より）
  - Workers の独自ランタイムによるベンダーロックインリスクは ADR-0001 で受け入れ済みであり、Cloudflare Bindings（D1 等）の採用はそのロックイン拡大と見なすことができる

### ADR-0002: フロントエンドフレームワーク選定
- 決定: React（Vite + React 19）を採用する。
- 今回の議題との関連: フロントエンドフレームワークの選定は、データベース選定と直接の技術的依存関係を持たない。フロントエンドはバックエンド API（Cloudflare Workers + Hono）を経由してデータにアクセスするため、データベースの選定はフロントエンド側に影響を与えない。
- 制約への影響（Certainty として扱うべき事項）: 特になし。フロントエンドとデータベースは API 層で完全に分離されており、React の採用がデータベース選定を制約することはない。
