# OBJ-5: 長期的なコスト安定性 — 評価結果

## 提案した代替案

- **Cloudflare D1**（ALT-1 として既提案）: Cloudflare Workers と同一プラットフォームの無料 SQLite DBであり、Free プランで 500 MB/DB・10 DB が提供される。個人規模のリーディングリスト（数千レコード）では無料枠内に収まりやすく、超過コストも読み書き単位の従量課金で予測しやすい。
- **Supabase**（ALT-2 として既提案）: 500 MB / 5万 MAU の無料枠を持つが、シングルユーザー用途では MAU 制限は実質問題なく、ストレージ上限が主な制約となる。組み込み認証機能があり追加コストを生みにくい構成が取れる。
- **Turso (libSQL)**（追加提案・既存代替案なし）: Cloudflare Workers との公式統合を持ち、Free プランで 5 GB・5 億ロウ読み取り/月と非常に広い無料枠を提供する。個人規模では Free プランを長期にわたって超過しにくい。ただし本 alternatives.md には含まれていない外部候補であるため参考扱いとする。

## 評価

### ALT-1: Cloudflare D1

- 評価: ○
- 利点:
  - Free プランは 500 MB/DB・10 DB。数千レコードのリーディングリスト（推定 1 MB 未満）では、長期にわたって無料枠内に収まる蓋然性が高い。
  - 超過時の課金単位はロウ読み取り/書き込み数であり、個人用途の軽量アクセスでは大幅なコスト増は起こりにくい。
  - Cloudflare のインフラサービス全体の一部であり、Workers・Pages との統合がビジネス上の理由から継続される公算が高い（プラットフォームとしての安定性）。
- 欠点:
  - 2025-02-10 に無料枠超過時の挙動がエラー返却に変更されており、制限変更の頻度・方向性に予見性がない（研究結果参照）。
  - Free プランの DB 数上限（10 DB）は今回は問題ないが、将来複数プロジェクトへの転用時に制約になる可能性がある。
- 根拠:
  - research.md 観点1より: D1 Free プランは 500 MB/DB・10 DB。GA 済みでプロダクション利用可能。
  - S4（D1 Free プランの継続提供は仮定）と D2（将来の制限変更リスク）に依存する部分がある。
- CSD依存:
  - S4（D1 Free プランが継続提供される）: 現時点では確認済みだが、将来にわたる保証はない。D2 の「無料枠縮小リスク」が顕在化した場合、この評価は △ に下がる可能性がある。
  - D2（D1 Free プランの将来的な縮小リスク）: 2025-02-10 の変更が示すように制限仕様は変更されうる。

---

### ALT-2: Supabase

- 評価: △
- 利点:
  - 無料枠は 500 MB ストレージ・5 万 MAU（月間アクティブユーザー）。シングルユーザー構成では MAU は実質 1 であるため、MAU 制限は問題にならない。
  - PostgreSQL の成熟したエコシステムが背景にあり、サービス自体の技術的信頼性は高い。
- 欠点:
  - **無料プロジェクトは 1 週間アクティビティなしで自動停止（一時停止）される**仕様がある。個人用途で利用頻度が低い期間が生じた場合、手動または API での再起動が必要となりコスト安定性に不確実性が生じる（公式ドキュメントより知られている仕様）。
  - Free プランはストレージ上限 500 MB であり、D1 と同等。ただしオブジェクトストレージ・Auth ユーザーデータ等も同一枠に含まれる可能性があり実質的な DB 用途上限は D1 より狭くなりうる。
  - Supabase は VC 資金調達型スタートアップであり、将来の料金体系変更リスクが D1（Cloudflare という大企業のインフラ部門）より相対的に高いと見なせる。
  - Paid プランへの移行コストは $25/月（Pro プラン）と D1 より高額。
- 根拠:
  - alternatives.md より: Supabase 無料枠は 500 MB・5 万 MAU。
  - Supabase の無料プロジェクト自動停止仕様は公知の制約。
- CSD依存:
  - S5 には Turso の継続仮定があるが Supabase に対応する仮定は明示されていない。Supabase の料金・無料枠継続については未分類の不確実性が存在する（潜在的な Doubt 項目）。

---

### ALT-3: TiDB Serverless

- 評価: ○
- 利点:
  - 無料枠は 5 GiB ストレージ・5,000 万 Request Units/月（alternatives.md より）。ストレージ上限は D1 の 10 倍であり、個人規模では実質的な制限になりにくい。
  - Request Units ベースの課金は軽量アクセスパターンに対して低コストであり、個人用途では超過しにくい。
  - TiDB は PingCAP という既存の商用データベース企業が提供しており、純粋スタートアップより事業継続性リスクが低い。
- 欠点:
  - TiDB Serverless の無料枠の「5,000 万 RU/月」という単位の粒度（1 クエリ = 何 RU か）が不明確であり、実際の消費量の予測可能性に不安がある。
  - 無料枠超過時の課金単価・上限設定が複雑になりやすい（RU ベースの従量課金）。
  - 日本市場での認知・事例が少なく、料金変更や無料枠廃止の予兆を察知しにくいリスクがある。
  - Cloudflare Workers からの接続は HTTP/TCP 経由の外部サービスとなり、接続オーバーヘッドが生じる（OBJ-1 との関連だが、コスト面では無関係）。
- 根拠:
  - alternatives.md より: 無料枠 5 GiB・5,000 万 RU/月。
  - RU 単位の粒度に関する予測可能性は research.md に明示情報がなく、不確実性が残る。
- CSD依存:
  - TiDB Serverless の無料枠継続に対応する仮定は CSD に存在しない。外部サービスとして S5（Turso の無料プラン継続）と類似した不確実性を持つ。

---

### ALT-4: Turso (libSQL)

- 評価: ○
- 利点:
  - Free プランはロウ読み取り 5 億回/月・ロウ書き込み 1,000 万回/月・ストレージ 5 GB・最大 100 DB（research.md 観点5より）。数千レコードのリーディングリストアプリの個人利用では、いずれの制限も長期にわたって超過する可能性がほぼない。
  - ストレージ上限は D1 の 10 倍（5 GB 対 500 MB）であり、将来のデータ増加に対して余裕がある。
  - 有料移行が必要になった場合の最安プランは Developer プラン $4.99/月（research.md 観点5より）と、Supabase Pro（$25/月）より低コストで、コスト増加が緩やかである。
  - クレジットカード不要で Free プランを開始でき、初期コスト障壁がない。
- 欠点:
  - Turso は独立した外部スタートアップ企業であり、Cloudflare（大企業インフラ部門）や PingCAP（既存商用 DB 企業）と比較して、事業継続リスク・料金体系の急変リスクが相対的に高い。
  - 無料枠の内容は 2025 年の Developer プラン新設時に見直された経緯があり（research.md 観点5より）、Free プランの条件が将来変更される可能性を否定できない。
  - Cloudflare エコシステム外のサービスであるため、Cloudflare 側のポリシー変更から保護されない反面、Cloudflare の料金変更リスクとは独立している。
- 根拠:
  - research.md 観点5より: Free プランは 5 億ロウ読み取り/月・1,000 万ロウ書き込み/月・ストレージ 5 GB・100 DB/アカウント。
  - research.md 観点5より: Developer プラン $4.99/月。
  - 個人規模（数百〜数千レコード）では読み取り・書き込み・ストレージのいずれも Free プラン上限に近づかないことが定量的に見積もれる。
- CSD依存:
  - S5（Turso の無料プランが継続提供される）: CSD に明示されている仮定。現時点では確認済みだが、Turso のビジネス状況によって将来変更される可能性がある。S5 が崩壊した場合、有料移行コストは Developer プラン $4.99/月が下限となる。
