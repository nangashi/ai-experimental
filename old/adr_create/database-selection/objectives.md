# 判断基準（Objectives）

## OBJ-1: データアクセスの応答性能

- 種別: fundamental（根本目的）
- 定義: データベースへのアクセスが、画面表示3秒以内の要件を安定して満たせること
- 評価の視点:
  - Cloudflare Workers（V8 Isolate）からの接続方式における接続オーバーヘッドの低さ（Binding vs HTTP API vs TCP）
  - データベースのリージョン・エッジとの物理的な距離・レイテンシ特性
  - コネクションプールなし環境（毎リクエスト接続確立）での現実的な応答時間

## OBJ-2: 認証機能の統合性

- 種別: fundamental（根本目的）
- 定義: ユーザー認証を低コストで実現できること。DBサービス自体が認証機能を提供しているか、またはWorkers側での認証実装とDBの統合がスムーズか
- 評価の視点:
  - DBサービスが提供する組み込み認証機能の有無・成熟度（Supabase Auth 等）
  - 認証を自前実装する場合のセッション/トークン管理ストレージとしての適性
  - 認証実装の追加工数・複雑さへの影響

## OBJ-3: バックアップ・リストアの実行可能性

- 種別: fundamental（根本目的）
- 定義: データの損失リスクに対して、現実的な手順でバックアップを取得・リストアできること
- 評価の視点:
  - エクスポート・インポート機能の有無と対応フォーマット
  - バックアップ手順の複雑さ（スクリプト化・自動化の容易さ）
  - リストア手順の確実性（FTS インデックス等の再構築要否を含む）

## OBJ-4: 運用負荷の最小化

- 種別: fundamental（根本目的）
- 定義: シングルユーザーの個人開発として、データベースのセットアップ・維持管理に要する労力が最小であること
- 評価の視点:
  - 初期セットアップの手順数・複雑さ
  - Cloudflare Workers との統合の容易さ（公式サポート・ドキュメントの充実度）
  - スキーマ変更・データ操作の手軽さ（CLI・管理ツールの存在）
  - 障害・トラブル発生時の対処のしやすさ

## OBJ-5: 長期的なコスト安定性

- 種別: fundamental（根本目的）
- 定義: 無料ティア内での継続利用が現実的に維持できること
- 評価の視点:
  - 現在の無料プランの制限内容（ストレージ上限・リクエスト数・DB数）
  - 無料プランの持続可能性（サービスの安定性・料金変更履歴）
  - 無料枠を超過した場合のコスト増加の予測可能性
