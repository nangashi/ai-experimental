# CSD分類

## 確実（Certainty）

- C1: データベースは Cloudflare Workers（V8 Isolate ランタイム）から接続可能な方式でなければならない — 根拠: ADR-0001 で確定。Workers は TCP ソケットの直接使用に制限があるため、HTTP/HTTPS ベースの API を持つデータベースサービス、または Cloudflare が公式に提供する Bindings（D1、KV 等）に限定される
- C2: 月額コストは 0 円（無料ティア内）でなければならない — 根拠: ADR-0001 の C3 で確定。有料プランへの移行は前提としない
- C3: シングルユーザー構成であり、大規模スケーリングへの対応は不要 — 根拠: ADR-0001 の C4 で確定。同時接続数・スループット要件は低い
- C4: データベースアクセスのレイテンシを含む画面表示 3 秒以内の要件がある — 根拠: ADR-0001 の C5 で確定。エンドツーエンドのレスポンスタイムに影響するため、データベース往復コストを考慮する必要がある
- C5: Cloudflare D1 は GA 済みでプロダクション利用可能な状態にある — 根拠: リサーチ（観点1）で確認。Free プランでも 500 MB/DB・10 DB が利用可能であり、数千レコードのリーディングリスト管理には十分

## 仮定（Supposition）

- S1: 管理するレコード数は将来的にも数千レコード（最大でも数万レコード）程度に収まる — 根拠: スコープに「数百〜数千レコード」とあるが、将来的なデータ増加は不確定。ただし個人向けリーディングリストという用途から大幅な増加は想定しにくい — 検証方法: 1年後にデータ件数・データサイズを実測し、D1 の Free プラン制限（500 MB）の 10% を超えていないか確認する
- S2: キーワード検索の要件は、タイトル・URL に対する部分一致検索で充足できる — 根拠: スコープに「タイトル・URLのキーワード検索」とある。現時点では LIKE 検索や FTS5 の全文検索で要件を満たせると想定するが、将来的な検索機能拡張（ベクトル検索、セマンティック検索）の可能性は排除できない — 検証方法: 実際の検索利用パターンを観察し、LIKE 検索または FTS5 で性能・機能的に十分かを確認する
- S3: バックアップ・リストアの用途は、定期的なスナップショット取得とデータ復元であり、頻繁な実行は不要 — 根拠: スコープに「バックアップ・リストア対応」とある。個人用途のため日次や週次程度の手動バックアップで十分と想定するが、要件の詳細は未確定 — 検証方法: バックアップ頻度・復元要件を利用者（自身）に確認する
- S4: Drizzle ORM は継続的にメンテナンスされ、Cloudflare D1 および Workers との互換性が維持される — 根拠: リサーチ（観点3）で Drizzle + D1 + Hono + Workers の組み合わせは実績があることを確認。ただしサードパーティライブラリのサポート継続は保証されない — 検証方法: Drizzle ORM の GitHub リポジトリの活動状況・D1 サポートのリリースノートを定期確認する
- S5: Turso の無料プランが継続して提供される — 根拠: リサーチ（観点5）で現時点の無料枠（5 GB、5億ロウ読み取り/月）を確認。ただしプライシング変更のリスクがある — 検証方法: Turso の料金ページを定期確認し、無料枠の縮小・廃止がないかモニタリングする

## 不確実（Doubt）

- D1: D1 の FTS5 仮想テーブルを含むデータベースがバックアップ（wrangler d1 export）の対象外であることの、実運用上の許容可能性 — 影響範囲: D1 を選択した場合のバックアップ・リストア対応。FTS5 仮想テーブル再作成手順の別途整備が必要 — リスク: 手順の複雑化によりバックアップ・リストアの確実性が低下する可能性。リストア後に FTS5 インデックスを再構築し忘れるリスク
- D2: Cloudflare D1 の Free プラン制限（500 MB/DB・10 DB/アカウント）が将来縮小または変更されるリスク — 影響範囲: D1 を選択した場合のコスト構造。現在は無料枠内だが、制限変更時に有料プランへの移行が必要になる — リスク: 無料枠超過時にエラーが返るようになったのが 2025-02-10 からと比較的最近であり、今後の制限変更の予測が難しい
- D3: Workers の Edge ロケーションと D1 データベースのリージョン間レイテンシが、画面表示 3 秒以内の要件に影響を与えるかどうか — 影響範囲: D1 を選択した場合のレスポンスタイム。Workers はグローバルに分散するが、D1 データベースは特定リージョンに固定される — リスク: ユーザーのロケーションが D1 リージョンから遠い場合、データベースアクセスのレイテンシが増加し、3 秒要件を超える可能性がある（ただし個人用途のため、実運用ユーザーは限定的）
