# OBJ-5: エコシステムの長期的安定性 — 評価結果

## 提案した代替案

- **React（Vite + React 19）**: npm ダウンロード数・GitHub スター数・バックにいる Meta 社のサポート・求人市場でのシェアなど、あらゆる規模指標で最大。破壊的変更の頻度は低く（React 16→17→18→19 の移行コストは軽微）、既存のチュートリアル・書籍・LLM 学習データが最も豊富であり、陳腐化リスクが最も低い。
- **Vue 3（Vite）**: Evan You と活発な OSS コミュニティに支えられ、Vue 2→3 移行（2022年）後は安定した 3.x 系が継続中。Stack Overflow Developer Survey での採用率は 17.6% で 2 位。Nuxt が Cloudflare Workers GA に含まれており、組織的なエコシステムの維持が見込める。
- **Svelte 5（SvelteKit）**: Svelte 5 は 2024年末に安定リリース済み。State of JS 2024「今後も使いたいフレームワーク」で最高スコア（62.4%）を記録し、開発者満足度は最高。ただし採用率 7.2%・既存学習リソースの量は React/Vue より少なく、Svelte 5 のルーンシステム導入に伴う既存リソースの陳腐化リスクがある。

## 評価

### React（Vite + React 19）

- 評価: ◎
- 利点:
  - npm 週次ダウンロード数が最大規模。Meta 社のバックアップに加え、Vercel・Shopify・Airbnb 等の大企業が本番採用しており、将来的なサポート打ち切りのリスクが極めて低い。
  - React 16→17→18→19 の移行コストはいずれも軽微。concurrent rendering・Server Components といった大機能追加でも後方互換性を維持してきた実績がある。
  - Stack Overflow、GitHub Issues、書籍、LLM 学習データのカバレッジが最大。今後 1-3 年で既存リソースが陳腐化するリスクが最も低い。
  - D2（チーム化の可能性）が現実化した場合でも、採用市場で React スキル保有者が最多であり、チーム参加障壁が低い。
- 欠点:
  - Server Components、React 19 の新 API（use フック・アクション）など、新機能が次々追加されており、「最新の React」に追従するための継続学習コストがかかる。ただし旧来の機能が即座に非推奨になることは少ない。
  - エコシステムの広さゆえ、ライブラリ選定（状態管理・ルーティング等）の判断コストが高い。
- 根拠: research.md 観点1「React は Stack Overflow Developer Survey で 44.7% の採用率、npm ダウンロード数・求人数ともに最大規模のコミュニティを維持」。React の直近のメジャー移行コストは軽微であったという業界認識に基づく。
- CSD依存: S5（メジャーバージョンの安定性）に軽微に依存。React 20 等の大規模変更が予告なく行われるシナリオは現時点の証拠からは想定しにくいが、完全な確実性ではない。

---

### Vue 3（Vite）

- 評価: ○
- 利点:
  - Vue 3.x 系は安定稼働中。Evan You 個人から OSS コミュニティ主体の体制に移行しており、単一スポンサー依存リスクは低減している。
  - Nuxt が Cloudflare Workers GA に含まれており、公式エコシステムの長期維持が期待できる。
  - 採用率 17.6% で 2 位を維持。書籍・チュートリアルの量は React に次ぐレベルで、今後 1-3 年での急激な縮小は考えにくい。
  - Vue 2→3 の大規模移行（Composition API 導入）をすでに完了しており、近い将来に同等規模の破壊的変更が発生する可能性は低い。
- 欠点:
  - D2（チーム化）が現実化した場合、React に比べ採用市場でのスキル保有者の母数が小さい。
  - React に比べ LLM 学習データのカバレッジが小さく、LLM 支援コーディングでの精度が若干劣る可能性がある（不確実）。
  - バックに大企業（Meta / Google 等）の直接支援がなく、コミュニティ主導である点はリスクとなりうる。
- 根拠: research.md 観点1「Vue.js は採用率 17.6% で 2 位、安定した地位を維持」。Cloudflare Workers GA に Nuxt が含まれることで公式エコシステムの連続性が確認できる。
- CSD依存: S5（メジャーバージョンの安定性）に依存。Vue 4 の大規模変更は現時点で兆候なし。D2（チーム化）が現実化した場合は採用市場規模の評価が変化する可能性がある。

---

### Svelte 5（SvelteKit）

- 評価: △
- 利点:
  - Svelte 5 は 2024年末に安定リリース済み。State of JS 2024 の満足度スコアが最高であり、開発者コミュニティの関心・熱量が高い。
  - Vercel（SvelteKit のメンテナー Rich Harris の雇用主）の支援があり、完全な個人開発プロジェクトではない。
  - 2025年10月時点でも継続的なアップデートが確認されており、プロジェクトが停滞するリスクは現時点では低い。
- 欠点:
  - 採用率 7.2% は React（44.7%）や Vue（17.6%）と比較して大幅に小さく、書籍・チュートリアル・LLM 学習データの量が限られる。D3（既存学習リソースの陳腐化リスク）と直接関連。
  - Svelte 5 のルーンシステム（`$state`, `$derived`, `$effect` 等）は Svelte 4 と大きく異なるシンタックスを導入しており、Svelte 5 以前のリソース（書籍・チュートリアル・Stack Overflow 回答の多数）は読み替えが必要になる。Svelte 4 の既存リソースが実質的に陳腐化している。
  - D2（チーム化）が現実化した場合、採用市場での Svelte スキル保有者の母数が著しく少なく、チーム参加障壁が高い。
  - React や Vue ほど大企業の直接的なプロダクション採用事例が多くないため、大規模な組織的サポートが薄い。
- 根拠: research.md 観点1・4。Svelte 5 の安定リリースは確認済みだが、ルーンシステム導入による既存リソースの陳腐化は OBJ-5 の「既存学習リソースの量と陳腐化リスク」に直接抵触する。
- CSD依存: D3（Svelte 5 ルーン導入後のドキュメント混在速度）に強く依存。Svelte 5 リソースが急速に充実するシナリオでは評価が ○ 方向に改善しうる。また D2（チーム化）が現実化した場合は × 方向にリスクが増大する。

---

### SolidJS（SolidStart 1.x）

- 評価: ×
- 利点:
  - SolidStart 1.0 は正式リリース済み。npm 週次ダウンロード数 約 108〜149 万件であり、完全な実験的プロジェクトではない。
  - シグナルベースのリアクティビティモデルはフロントエンド業界全体への影響があり、将来的な汎用知識になりうる。
- 欠点:
  - 採用率 2% 未満。npm ダウンロード数は React の 1/10〜1/20 程度であり、コミュニティ規模・学習リソースの量において他の候補と圧倒的な差がある。
  - SolidStart 2.0.0-alpha が開発中であり、1.x 系と 2.x 系で API の大幅な変化が予想される。D3 と同等の「ドキュメント混在」リスクが現時点でも発生している。
  - バックに大手企業のサポートがなく、個人主導のコミュニティプロジェクトに近い構造であり、長期的な開発継続への保証が最も薄い。
  - Cloudflare Workers サポートが 2025 Q2 時点でベータ段階であり、C1（Cloudflare Pages への静的デプロイ）は問題ないが、将来的な構成変更時のリスクが残る。
- 根拠: research.md 観点5「SolidJS は採用率 2% 未満、エコシステム・学習リソースの規模において他の主要フレームワークと大きな差がある」「SolidStart 2.0 への移行期にあるため、学習リソースやコミュニティサポートが 1.x と 2.x で混在するリスク」。
- CSD依存: D3（SolidStart 2.0 の API 変更による既存リソースの陳腐化速度）に強く依存。S5（メジャーバージョンの安定性）についても SolidStart は他の候補より崩れやすい仮定として扱うべき。
