# OBJ-3: 新しい概念・パターンの学習価値 — 評価結果

## 提案した代替案

この目的（既存スキルと異なるアプローチの学習）を最適化する観点から、以下の代替案が適している。

- **Svelte / SvelteKit**: Reactの仮想DOMとコンポーネントモデルとは根本的に異なるコンパイル時リアクティビティモデルを採用。ランタイムライブラリをほぼ持たない「コンパイラとしてのフレームワーク」という設計思想は、Reactとは全く異なる概念を学べる。Web標準（HTML/CSS/JS）に近い構文でありながら、リアクティビティを宣言的に記述するパターンは汎用性が高い。
- **SolidJS**: Reactに似た JSX 構文を持ちながら、仮想DOMを使わないシグナルベースのリアクティビティシステムを採用。Reactの知識を出発点として、より低レイヤーなリアクティビティの概念を深く学べる。シグナルは Angular Signals / Vue Composition API などとも共通する概念であり、フレームワーク横断の汎用性がある。
- **Vue 3 (Composition API)**: Reactのカスタムフックと概念的に類似した Composition API を採用しているが、リアクティビティシステム（Proxy ベース）・SFC（Single File Component）の構造・テンプレート構文はReactと異なる。エコシステムの充実度とReactとのコンセプト的な近さのバランスが取れており、学習コストを抑えながらも新概念を体験できる。

## 評価

### React (Vite + React)

- 評価: ×
- 利点:
  - 開発者がすでに持つReact知識を深める（Hooks、Concurrent Modeなど）ことで既存スキルを高度化できる。
  - 最新のReact 19のサーバーコンポーネント・Server Actionsという新概念はあるが、今回はCloudflare Pages静的構成のため、これらの新機能は実質使用しない。
- 欠点:
  - 既存スキル（React）と異なるアプローチやアーキテクチャパターンの学習機会がほぼない。
  - 仮想DOMモデル・コンポーネントライフサイクルの考え方はすでに習得済みであり、新しい設計思想の体験可能性が低い。
  - Web標準に近い書き方の習得という視点でも、JSXはWeb標準（HTML）からやや乖離している。
  - 習得した知識の他フレームワークへの転用性はReact固有スキルの深化であり、OBJ-3が求める「異なるアプローチの習得」ではない。
- 根拠: OBJ-3の定義は「既存スキル（React）と異なるアプローチやアーキテクチャパターンを学習できること」であり、Reactを使い続けることはこの目的を満たさない。
- CSD依存: S1（開発者はReact知識を持つが他フレームワーク経験はない）に依存。S1が崩れる（開発者がすでに他フレームワーク経験を持つ）場合、評価が変わりうる。

---

### Vue 3 (Vite + Vue)

- 評価: ○
- 利点:
  - Reactとは異なるリアクティビティモデル（Vue 3のProxyベースリアクティビティ）を学べる。
  - SFC（Single File Component）の構造（`<template>` / `<script>` / `<style>` の単一ファイル分離）はReactとは異なるコンポーネント設計パターンであり、Web標準に近いHTMLテンプレート構文を体験できる。
  - Composition APIはReactのカスタムフックと概念的に近く、Reactユーザーにとって移行コストが低い一方、実装レベルでの仕組みの違い（ref / reactive / watchEffect）を学べる。
  - 習得したVueのリアクティビティの考え方（ref/reactive）はSvelteのstoreや他フレームワークの理解にも転用可能性がある。
  - 公式チュートリアル（vuejs.org）の質が高く、日本語・英語ともに充実している。
- 欠点:
  - Reactとの概念的距離はSvelte・SolidJSよりも小さく、「全く異なる設計思想」の体験という点ではやや弱い。
  - Options APIはReact開発者には逆に混乱を招く可能性があるが、Composition APIに絞れば問題は少ない。
  - コンパイル時最適化・シグナルベースといった最先端のリアクティビティパターンを体験する機会は限定的。
- 根拠: Reactユーザーが「異なるが近い」概念を学ぶには適しているが、OBJ-3の評価視点「Reactと異なる設計思想の体験可能性」の観点でSvelteやSolidJSより一段低い。
- CSD依存: S1（開発者のReact経験）に依存。D1（学習目的の優先度）が高い場合はより差別化が重要になる。

### Svelte 5 / SvelteKit

- 評価: ◎
- 利点:
  - **コンパイラとしてのフレームワーク**という根本的に異なる設計思想を体験できる。ReactのようなランタイムVirtualDOMではなく、コンパイル時にリアクティビティをバニラJSに変換するアプローチは、フロントエンドアーキテクチャの本質的な違いを学べる。
  - **Svelte 5のRunesシステム**（`$state`, `$derived`, `$effect`）はシグナルベースのリアクティビティの現代的な実装であり、Angular Signals・Preact Signals・Vue 3 reactivityなど他フレームワークの方向性と共鳴する汎用的な概念を体験できる。
  - **Web標準に近い構文**：`<script>`タグ内に直接宣言的なコードを書くスタイルは、HTMLテンプレートとの親和性が高く、「Webの基礎」を再認識できる。
  - 習得した概念（コンパイラによるリアクティビティ、シグナルパターン）は今後の業界トレンドに沿っており、転用価値が高い。
  - 公式チュートリアル（learn.svelte.dev）はインタラクティブで質が高い。Svelte 5の日本語・英語学習リソースは2025年以降急増している（research.md: Svelte 5は2024年末安定リリース済み）。
- 欠点:
  - Svelte 4→5での構文変更（Runesへの移行）により、Stack OverflowやLLMの既存の学習データにSvelte 4時代の旧構文が混在している。質問時に回答が旧構文（`$:`リアクティブ宣言など）で返される可能性がある。
  - SvelteKit固有の概念（ルートベースのload関数、form actions）はReactエコシステムとは全く異なるため、最初の立ち上げに時間を要する可能性がある。
  - コミュニティ規模はReactに比べ小さく、エラーに遭遇した際の検索ヒット数が少ないシナリオがありうる。
- 根拠: OBJ-3の全評価視点を高いレベルで満たす。コンパイル時リアクティビティ・Runesによるシグナルパターン・Web標準に近い構文・転用可能な汎用概念・高品質な公式チュートリアルのすべてが揃っている。
- CSD依存: S4（vite-plugin-pwaがSvelteKitで安定動作するか）に部分的に依存するが、OBJ-3自体はS4の影響を受けない。S5（Svelteのメジャーバージョン安定性）は満たされている（Svelte 5は安定リリース済み）。D1（学習目的の優先度）が高い場合に評価がより際立つ。

### SolidJS / SolidStart

- 評価: △
- 利点:
  - **シグナルベースのリアクティビティの純粋な実装**：SolidJSのシグナル（`createSignal`, `createEffect`, `createMemo`）は、リアクティビティの最も根本的な概念を学べる教材として優れている。ReactのHooksとは異なる依存追跡の仕組みを体験できる。
  - **仮想DOMなしのJSX**：JSX構文でありながら仮想DOMを使わないアプローチは、「JSX = React」という思い込みを解体できる。コンポーネントが一度だけ実行されるという設計は、Reactのレンダリングモデルとの対比で深い理解を生む。
  - シグナルパターンはフレームワーク横断で汎用的な概念（Angular Signals、Vue Reactivity、Svelte Runesとも共通）であり、転用価値が高い。
- 欠点:
  - **SolidStart 1.x → 2.0 移行期のドキュメント混在リスク**（research.md: SolidStart 2.0-alpha 開発中）。学習目的での採用において、旧バージョンの情報と新バージョンの情報が混在し、学習効率が著しく低下するリスクがある（D3と直接対応）。
  - **エコシステム規模の小ささ**（npm週次ダウンロード約108〜149万件、Reactの1/10〜1/20）により、LLMの学習データへの網羅度・Stack Overflowの情報量がReact/Vue/Svelteと比べて大幅に少ない。詰まったときの解決情報が得られにくい。
  - 公式チュートリアルの質は高いが、日本語リソースは限定的。
  - 採用事例・実装パターンの参考資料が少なく、学習リソース（OBJ-3の評価視点「学習リソースの質と現時点での最新性」）の評価では弱い。
- 根拠: 概念的な学習価値は高いが、SolidStart 2.0移行期のドキュメント混在リスク（D3）とエコシステムの小ささが、学習体験の質を損なう可能性がある。学べる概念自体は魅力的だが、「学習リソースの質と最新性」「転用可能な知識の汎用性」の観点でSvelteに劣る。
- CSD依存: D3（SolidStart 2.0のAPI変更による学習リソース陳腐化速度が不明）に強く依存。D3が悪化した場合（2.0リリースが早まり大規模な変更が発生）、評価がさらに低下する。D3が解消（1.x系が長期維持される）した場合、△から○への引き上げが考えられる。
