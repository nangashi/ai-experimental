## 各セルの詳細

### ALT-1 × OBJ-1（○）
- 評価: ○
- 利点: vite-plugin-pwa の事例・情報量が最も豊富。トラブル時の解決情報が入手しやすい
- 欠点: 情報量以外の実質的優位性はなく、PWA実装難易度自体はフレームワーク非依存
- 根拠: research.md より vite-plugin-pwa は全 Vite フレームワークを統一サポート。React の優位性は情報量のみ

### ALT-1 × OBJ-2（◎）
- 評価: ◎
- 利点: 既存 React スキル（S1）により学習コストゼロ。エコシステム・問題解決情報が全候補中最大
- 欠点: OBJ-3（学習価値）とのトレードオフ
- 根拠: S1 に基づき、開発者の既存スキルと完全に連続する唯一の選択肢

### ALT-1 × OBJ-3（×）
- 評価: ×
- 利点: React 19 の新機能で既存スキル深化は可能
- 欠点: 既存スキルと異なるアプローチの学習機会がほぼない
- 根拠: OBJ-3 の定義「既存スキルと異なるアプローチの学習」を本質的に満たさない

### ALT-1 × OBJ-4（○）
- 評価: ○
- 利点: Vite SPA として CF Pages への静的デプロイが完全に確立。追加アダプター不要
- 欠点: SSR 移行時に React Router v7 / OpenNext 経由が必要でフレームワーク変更を伴う
- 根拠: S6（SSR移行可能性）に依存。SSR 不要なら ◎、早期に必要なら △

### ALT-1 × OBJ-5（◎）
- 評価: ◎
- 利点: 最大エコシステム。Meta 支援。破壊的変更の実績が少ない。LLM学習データカバレッジ最大
- 欠点: ライブラリ選定の判断コストが高い
- 根拠: 採用率 44.7%、npm ダウンロード数最大。陳腐化リスクが最も低い

### ALT-2 × OBJ-1（○）
- 評価: ○
- 利点: vite-plugin-pwa 対応済み。事例数は React に次ぐ規模
- 欠点: React と比較して情報量が若干少ない
- 根拠: React との差は情報量の程度差。実装難易度に本質的な差はない

### ALT-2 × OBJ-2（○）
- 評価: ○
- 利点: Composition API は React hooks に類似。React 経験者の移行コストが低い。日本語ドキュメント充実
- 欠点: テンプレート構文の学習が初期に必要。React ほどの問題解決情報量はない
- 根拠: 1-2 日で基本的な CRUD が書けるようになると想定

### ALT-2 × OBJ-3（○）
- 評価: ○
- 利点: Proxy ベースリアクティビティ・SFC 構造は React と異なる概念を学べる
- 欠点: React との概念的距離が Svelte より小さい
- 根拠: 「異なるが近い」概念の学習には適しているが、根本的な設計思想の違いは体験しにくい

### ALT-2 × OBJ-4（○）
- 評価: ○
- 利点: Vite SPA として CF Pages 静的デプロイが確立。Nuxt が CF Workers GA
- 欠点: SSR 移行時は Nuxt への追加学習が必要
- 根拠: React の Vite 構成と同等の安定性。SSR 移行パスは React より明確（Nuxt GA）

### ALT-2 × OBJ-5（○）
- 評価: ○
- 利点: 安定した 3.x 系。採用率 17.6% で 2 位。Nuxt が CF Workers GA
- 欠点: React に比べ採用市場・LLM データカバレッジが小さい。大企業の直接支援なし
- 根拠: Vue 2→3 の大規模移行を完了済み。近い将来の同規模変更は低確率

### ALT-3 × OBJ-1（○）
- 評価: ○
- 利点: @vite-pwa/sveltekit 専用パッケージ。SvelteKit の navigation preloading が PWA 体験を補完
- 欠点: 専用パッケージの設定が React/Vue より複雑。Svelte 5 後の統合事例がまだ少ない
- 根拠: S4（@vite-pwa/sveltekit の安定動作）に依存

### ALT-3 × OBJ-2（△）
- 評価: △
- 利点: Svelte 構文は HTML/CSS/JS に近い。Svelte 5 Runes は React hooks に近い設計
- 欠点: 実務経験なし（S1）。Svelte 5 ドキュメント混在。SvelteKit のファイル構造が複雑。採用率 7.2% で情報量が少ない
- 根拠: 初期学習コストが React・Vue より高い。立ち上げ直後の数週間に学習投資が集中

### ALT-3 × OBJ-3（◎）
- 評価: ◎
- 利点: コンパイル時リアクティビティという根本的に異なる設計思想。Runes で現代的シグナルパターン。Web 標準に近い構文
- 欠点: Svelte 4 との構文変更で旧情報が混在しうる
- 根拠: 全評価視点を高いレベルで満たす。転用可能な汎用概念の習得に最適

### ALT-3 × OBJ-4（◎）
- 評価: ◎
- 利点: @sveltejs/adapter-cloudflare で公式対応。CF Workers GA 認定。静的/SSR をアダプター切替のみで対応可能
- 欠点: SvelteKit 専用ビルド設定が必要。アダプター依存
- 根拠: Cloudflare 公式ドキュメントにデプロイガイド存在。SSR 移行が最もシームレス

### ALT-3 × OBJ-5（△）
- 評価: △
- 利点: Svelte 5 安定リリース済み。開発者満足度最高。Vercel の支援
- 欠点: 採用率 7.2%。Svelte 5 Runes で既存リソース陳腐化。D2（チーム化）時にスキル保有者が少ない
- 根拠: D3（ドキュメント混在速度）に依存。リソース充実が進めば ○ 方向に改善しうる

### ALT-4 × OBJ-1（△）
- 評価: △
- 利点: vite-plugin-pwa は SolidJS にも対応
- 欠点: SolidStart 移行期でドキュメント混在。エコシステムが小さく事例が乏しい
- 根拠: D3 に強く依存。情報不足・移行期リスクが重なる

### ALT-4 × OBJ-2（×）
- 評価: ×
- 利点: JSX 構文で React 経験者に表面的な親和性あり
- 欠点: リアクティビティモデルの根本的な違いで React 経験が「負の転用」に。SolidStart 移行期。エコシステム最小
- 根拠: 問題解決コストが最も高い。D3 が悪化すると著しいリスク

### ALT-4 × OBJ-3（△）
- 評価: △
- 利点: シグナルベースのリアクティビティの純粋な実装。仮想 DOM なし JSX で概念を解体
- 欠点: SolidStart 移行期のドキュメント混在が学習体験を損なう。エコシステムが小さく学習リソース不足
- 根拠: 概念的学習価値は高いが D3 と情報不足で実際の学習体験が損なわれる

### ALT-4 × OBJ-4（△）
- 評価: △
- 利点: start-cloudflare-pages アダプター経由でデプロイ可能
- 欠点: CF Workers サポートがベータ段階。SolidStart 2.0 で設定変更の可能性。CF Workers GA 一覧に未掲載
- 根拠: D3 に依存。S5 が SolidStart では成立しにくい

### ALT-4 × OBJ-5（×）
- 評価: ×
- 利点: SolidStart 1.0 は正式リリース済み。シグナルは業界トレンド
- 欠点: 採用率 2% 未満。大手企業サポートなし。SolidStart 2.0 移行期で API 変化予想
- 根拠: コミュニティ規模・学習リソース量で他候補と圧倒的な差
