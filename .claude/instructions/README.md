# Instructions

CLAUDE.md の Instructions テーブルに登録され、条件合致時に自動読み込みされる指示ファイル群。
主な読み手は LLM であり、各エントリの情報から適用判断と行動決定を行う。

## 追加基準

エントリを新規追加する前に以下を満たすことを確認する。該当しない場合は既存エントリの修正・拡張で対応する。

1. scopeの適用範囲が狭すぎず広すぎず、適切な粒度であること。特定の1タスクにしか適用できないものは汎用性が不足し、常時該当するものは判定コストだけが増える
2. scopeが具体的であり、LLMが適用の判断を行いやすいこと
3. actionsがAIの行動の安定した制約に繋げられる内容であること。実行者が変わっても同じ行動になるかを判断基準とする
4. 既存エントリと実質的に同じ状況・行動を対象としていないこと。追加前に既存エントリのscope・actionsと照合し、以下を確認する:
   - 同じ状況に対して同じ行動を別の表現で述べていないか（重複）
   - 同じ状況に対して矛盾する行動を指示していないか（矛盾）
   - 既存エントリのactionsに1項目追加すれば済む内容ではないか（統合候補）

### 配置先の判断

追加基準を満たしたエントリは、scopeが既存ファイルのuse-when条件に合致するファイルに配置する。合致するファイルがない場合は、新ファイル作成の要否を判断する。

## エントリ構造

各エントリは以下のフィールドで構成する。

| フィールド | 必須 | 形式 | 役割 |
|-----------|------|------|------|
| **title** | ○ | `##` 見出し | エントリの識別。原則の内容を端的に表す |
| **scope** | ○ | 散文 | 適用条件。LLM がこのエントリを適用すべき状況を示す |
| **actions** | ○ | 箇条書き | 具体的な行動指示 |
| **rationale** | ○ | 散文 | 因果メカニズムの説明。重要度・影響度の情報を含めてよい |
| **condition** | - | 散文 | 例外・注意事項。該当する場合のみ記載 |
| **sources** | - | 箇条書き | 根拠となる外部文献へのリンク。基本1件。根拠として不十分なときは補足として1件追加してよい。プロジェクト内ファイルは不可 |

### フィールドの判断フロー

```
1. この原則は今の状況に該当するか？ → scope
2. 何をすべきか？                   → actions
3. なぜこれが有効なのか？           → rationale（一般化・応用判断に使用）
4. 例外や注意すべき状況はあるか？   → condition
```

### title の形式

title は `##` 見出しとして記載する。エントリの区切りと識別を兼ねる。

## ファイル構成

| ファイル | 内容 |
|---------|------|
| `workflow-design.md` | ワークフローの構造設計・コンテキスト管理・データフロー |
| `prompt-design.md` | プロンプト設計のガイドライン（分解・テクニック選択・バイアス回避） |
| `codings/coding-antipatterns.md` | コード生成時のセルフチェック（デッドコード・設計原則の適用境界・過剰抽象化） |
| `examples/` | 実装パターン集。本体から参照される補足資料 |
