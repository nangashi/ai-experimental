以下の手順で固定テスト対象文書と正解キーを生成してください:

1. Read で以下のファイルを読み込む:
   - {test_document_guide_path} （テスト対象文書生成ガイド — 入力型判定と文書構成を含む）
   - {perspective_path} （観点定義 — 問題バンクを含まない作業コピー）
   - {perspective_source_path} （観点定義ソース — 問題バンクを含む。テスト文書の問題埋め込み時に参考にする）

2. test-document-guide.md のセクション1に従い、perspective.md の概要からエージェントの入力型を判定する

3. 入力型に応じたテスト対象文書を生成する
   - ドメインは perspective のスコープに適したものを自由に選択する
   - **分量目安**: 200-300行（設計書の場合）。15問を自然に埋め込める十分な量を確保する

4. 以下のパラメータで問題を埋め込む:

   | 項目 | 基準 |
   |------|------|
   | **総数** | 15問 |
   | **深刻度分布** | 重大 4-5個 / 中 6-7個 / 軽微 4-5個 |
   | **カテゴリ分布** | 指定観点の主要問題 8-9個 + 隣接領域 3-4個 + 微妙な問題 2-3個 |
   | **検出難易度分布** | easy 3-4個 / medium 6-8個 / hard 3-5個 |
   | **カテゴリ最低配分** | perspective の評価スコープの各カテゴリに最低2問を配分すること |

   **禁止パターン**（test-document-guide.md の「禁止パターン」に該当する埋め込みは不可）:
   - 括弧内で弱い判断を正当化する記述
   - 明示的にアンチパターンを推奨する記述
   - 問題を目立たせるための機密データの不自然な列挙

   **推奨**: test-document-guide.md の「埋め込みの具体的技法」から問題ごとに適切な技法を選択し、5種のうち少なくとも3種を使用する

   test-document-guide.md の「問題埋め込みガイドライン」の原則に従う:
   - 自然さ: 上記の禁止パターンを避け、test-document-guide.md の具体的技法を使用する
   - 多様性: 同じカテゴリでも異なるパターンを使う
   - 深さのバリエーション: 表面的な問題と深い分析が必要な問題を混在
   - 文脈依存: 他セクションとの関連で問題になるケースを含める
   - 隣接領域: スコープ外だが検出できると有益な問題を含める

5. 正解キーを生成する。test-document-guide.md のセクション4のフォーマットに従い:
   - 各問題に ID (P01-P15)、カテゴリ、深刻度、検出難易度、該当箇所、検出判定基準 (○/△/×の条件) を記載
   - 検出難易度は test-document-guide.md の定義に従い easy/medium/hard を設定する
   - ボーナス問題リスト (B01-B08程度) を含める

6. Write で以下を保存する:
   - {test_document_save_path}
   - {answer_key_save_path}
