以下の手順でテスト対象文書と正解キーの品質を検証してください:

## 入力の読み込み

1. Read で以下のファイルを読み込む:
   - {test_document_path} （テスト対象文書）
   - {answer_key_path} （正解キー）
   - {perspective_path} （観点定義 — 問題バンクを含まない作業コピー）

## 検証1: テスト対象文書の自然さ

2. テスト対象文書を**正解キーを参照せずに**通読し、以下の不自然パターンを確認する:

   **不自然パターン（1つでも該当すればFAIL）**:
   - **括弧内正当化**: 弱い設計判断の直後に括弧で防御的な理由を述べている
   - **明示的アンチパターン推奨**: セキュリティ上明らかに悪い慣行を意図的に採用している記述
   - **機密データの不自然な列挙**: 問題を目立たせるためにセンシティブな要素を不自然に並べた記述

## 検証2: 正解キーの検出判定基準の明確性

3. 各問題の検出判定基準を確認する:
   - ○と△の境界が一意に解釈可能か
   - △の基準が「関連カテゴリへの言及はあるが、{核心}に触れていない」形式か
   - 「適切に指摘している」等の主観的表現がないか

## 検証3: 分布要件の充足

4. 以下の分布要件を確認する:

   | 項目 | 要件 | 実際 | 判定 |
   |------|------|------|------|
   | 総数 | 10問 | {N} | OK/NG |
   | 深刻度: 重大 | 3 | {N} | OK/NG |
   | 深刻度: 中 | 4 | {N} | OK/NG |
   | 深刻度: 軽微 | 3 | {N} | OK/NG |
   | 検出難易度: easy | 2-3 | {N} | OK/NG |
   | 検出難易度: medium | 4-5 | {N} | OK/NG |
   | 検出難易度: hard | 2-3 | {N} | OK/NG |

## 検証4: perspectiveカテゴリのカバレッジ

5. perspective の「評価スコープ」の各カテゴリに最低1問が配分されているか確認する

## 結果の返答

6. 以下のフォーマットで返答する:

```
## 検証結果サマリ
- 自然さ: {OK / 問題あり（N箇所）}
- 判定基準の明確性: {OK / 要改善（N問）}
- 分布要件: {OK / NG（項目名）}
- カテゴリカバレッジ: {OK / 欠落あり（カテゴリ名）}
- 総合判定: {PASS / FAIL}

## 問題詳細（FAILの場合のみ）
{問題の詳細リスト}
```
