# 観点定義: 盲点検出分析（Blind Spots Detection）

## 概要

エージェントの評価履歴データから系統的な検出失敗（盲点）を特定する分析サブエージェント（analyze-blind-spots）のテスト用観点。テストエージェント定義＋シミュレート済み履歴データに埋め込む「盲点分析の品質問題」のカテゴリと例を定義する。

テスト対象: analyze-blind-spots サブエージェントが、履歴データ中の盲点パターンをどれだけ正確に検出・分類できるか。

**特記**: このサブエージェントのテストには `test_type: "definition+history"` が必要。テストエージェント定義に加え、シミュレートされた複数ラウンドのスコアリングデータを生成する。

## 評価スコープ

analyze-blind-spots サブエージェントが検出すべき問題:

- **検出率集計の正確性**: カテゴリ別の検出率（○/△/×）を正しく集計できるか
- **盲点分類の正確性**: persistent blind spot（<50%検出）とstructural blind spot（0%検出）を正しく分類できるか
- **ルート原因の特定**: 盲点の原因（基準欠落/知識不足/指示不足）を正しく特定できるか
- **偽陽性盲点の回避**: 実際には検出できているカテゴリを盲点と誤報告しないか
- **データ不足時の適切な判断**: データが2ラウンド未満の場合に分析を控えるか

## スコープ外

- 評価基準の品質問題 → criteria-effectiveness で扱う
- スコープ定義の問題 → scope-alignment で扱う
- ドメイン知識の不足 → domain-knowledge-adequacy で扱う
- 盲点以外のパフォーマンス問題（スコア全体の高低ではなく、カテゴリ別の系統的失敗のみ対象）

## テストデータ要件

generate-test-agent.md が `test_type: "definition+history"` で呼ばれた場合、以下も生成する:
- シミュレートされた `scores.md`（2-3ラウンド分）
- シミュレートされたスコアリングファイル（各ラウンドの○/△/×詳細）
- これらのデータに意図的な盲点パターンを埋め込む

## ボーナス/ペナルティの判定指針

- **ボーナス対象**: 正解キーに含まれないが、データから導出可能な有効な盲点指摘
- **ペナルティ対象**: データに根拠のない推測的盲点報告、スコープ外の指摘

## 問題バンク

| カテゴリ | 問題例 | 深刻度 |
|---------|--------|-------|
| 構造的盲点の見逃し | 全ラウンドで検出率0%のカテゴリが存在するが、報告されない | 重大 |
| 偽陽性盲点 | 検出率80%のカテゴリを「盲点」と誤分類（△を×と混同） | 重大 |
| データ不足での過剰推論 | 1ラウンドのデータで「構造的盲点」と断定（最低2ラウンド必要） | 重大 |
| 永続的盲点の分類ミス | 検出率30%（persistent）を検出率0%（structural）と誤分類 | 中 |
| ルート原因の誤特定 | 実際は指示の曖昧さが原因なのに「基準欠落」と報告 | 中 |
| 検出率計算ミス | ○が2/5なのに検出率を50%ではなく40%と報告（計算エラー） | 中 |
| カテゴリ粒度の不適切さ | 細分化すべきカテゴリを一括集計し、部分的盲点を見逃す | 中 |
| 改善推奨の欠落 | 盲点を特定するが、具体的な改善推奨がない | 軽微 |
| 重複盲点の未統合 | 同じ原因に起因する複数の盲点を個別に報告（統合すべき） | 軽微 |
| トレンド分析の欠如 | ラウンド間の改善/悪化トレンドに言及しない | 軽微 |
