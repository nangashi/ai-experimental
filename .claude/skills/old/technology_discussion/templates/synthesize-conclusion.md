# 結論統合テンプレート

複数ラウンドの議論結果を統合し、結論ドラフトを生成する。

## 入力変数

- `{topic}`: 議題（テキスト）
- `{work_dir}`: 作業ディレクトリの絶対パス
- `{discussion_points_path}`: 論点ファイルの絶対パス
- `{draft_save_path}`: 結論ドラフトの保存先絶対パス
- `{personalities}`: 参加した3性格名（カンマ区切り。例: pragmatist,skeptic,idealist）

## 手順

1. 以下のファイルを Read で読み込む:
   - `{discussion_points_path}` — 論点一覧
   - `{work_dir}/round1-*.md` — Round 1 の各性格の意見（Glob で検出）
   - `{work_dir}/round2-*.md` — Round 2 の反論（存在する場合のみ）
   - `{work_dir}/round3-*.md` — Round 3 の直接議論結果（存在する場合のみ）
   - `{work_dir}/conflicts.md` — 対立点整理（存在する場合のみ）

2. 各論点について、以下を分析する:
   - 各性格がどの立場を取ったか
   - 議論を通じて意見がどう変化したか
   - 合意点と残存する対立点

3. 結論を導出する際の原則:
   - 3性格のうち2つ以上が一致する論点は、その方向で結論を出す
   - 全員が異なる立場の論点は、根拠の強さと具体性で判断する
   - 「やらない理由」より「やる理由」が具体的で検証可能な場合はやる方向に倒す
   - リスク指摘（Skeptic）が具体的で重大な場合は、軽減策を条件に結論を出す

4. `{draft_save_path}` に以下のフォーマットで Write する:

```markdown
# {topic} — 結論ドラフト

## 結論
（採用方針の明確な記述）

## 根拠
### 論点1: {タイトル}
- 判断: {採用した方向}
- Pragmatist の意見: {要約}
- Skeptic の意見: {要約}
- Idealist/Connector の意見: {要約}
- 判断理由: {なぜこの方向にしたか}

（各論点について同様に記載）

## 却下された代替案
### {代替案1}
- 提唱者: {性格名}
- 内容: {概要}
- 却下理由: {具体的理由}

## 議論のポイント
（特に意見が分かれた論点と、その議論の経過）

## 留意事項
（結論を実行する際のリスク・注意点。特にSkepticの指摘から）
```

## 返答フォーマット

1行目: 結論の1文サマリ
2行目: 合意論点数/全論点数

詳細は全て `{draft_save_path}` に記載すること。
