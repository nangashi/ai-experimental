# 情報収集テンプレート

議題に関連する技術的背景とコードベース情報を収集し、議論の土台となる調査レポートを生成する。

## 入力変数

- `{topic}`: 議題（テキスト）
- `{work_dir}`: 作業ディレクトリの絶対パス
- `{research_save_path}`: 調査結果の保存先絶対パス

## 手順

1. 議題 `{topic}` を分析し、調査すべき技術的側面を特定する（3-5個）

2. コードベース調査:
   - Glob と Grep で議題に関連するファイル・コード・設定を検索する
   - 既存の実装パターン、使用中の技術スタック、依存関係を確認する
   - 関連ファイルが見つかった場合は Read で内容を確認する
   - 該当するコードがない場合は「コードベースに関連する既存実装なし」と記録する

3. 技術的背景調査:
   - WebSearch で議題に関連する技術的な比較情報・ベストプラクティス・既知の問題点を調査する
   - 検索は2-3回に限定し、最も関連性の高い情報を抽出する

4. 調査結果を `{research_save_path}` に以下のフォーマットで Write する:

```markdown
# 調査レポート: {topic}

## コードベース状況
（既存の関連実装・設定・依存関係。なければ「関連する既存実装なし」）

## 技術的背景
（各選択肢の特徴・メリット・デメリットの客観的整理）

## 業界動向
（採用事例・トレンド・既知の問題点）

## 制約・前提条件
（コードベースから読み取れる制約。例: 使用中のフレームワーク、デプロイ環境等）
```

## 返答フォーマット

1行目: 調査完了を示すサマリ（調査した側面の数と主要な発見を1文で）

詳細は全て `{research_save_path}` に記載すること。
