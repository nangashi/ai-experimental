以下の手順でPR更新用のコンテンツを生成してください:

## 手順

### 1. 設計の把握

{design_path} を Read で読み込み、「概要」セクションを把握する。

### 2. 実装計画の把握

{plan_path} を Read で読み込み、全ステップの概要を把握する。

### 3. レビュー結果の把握

{consolidated_path} を Read で読み込み、レビュー結果のサマリを把握する。

### 4. PR追記コンテンツの生成

以下の2つのセクションを生成し、{pr_body_save_path} に Write で保存する:

```markdown
## 実装サマリ

### 変更概要
{設計の概要セクションをもとに、実装した内容を1-3文で要約}

### 実装ステップ
{各ステップの名前と変更概要を箇条書きで列挙}

### 自動レビュー結果
- 品質: {pass/needs_revision + 指摘数}
- テスト: {pass/needs_revision + 指摘数}
- セキュリティ: {pass/needs_revision + 指摘数}
- 設計適合: {pass/needs_revision + 指摘数}
- 統合判定: {pass/needs_revision}

---

## レビュー依頼

### 注目ポイント
{レビュアーが特に確認すべき箇所を2-3点。複雑なロジック、重要な設計判断、自動レビューで Moderate の指摘があった箇所など}

### 変更規模
- 新規ファイル: {数}
- 変更ファイル: {数}
- 実装ステップ: {数}

### 確認済み事項
- 自動レビュー（品質・テスト・セキュリティ・設計適合）通過
- lint/format 通過
- テスト通過
```

### 5. 返答

以下のフォーマットで返答する:

```
result: success
summary: {生成したコンテンツの1行サマリ}
```
