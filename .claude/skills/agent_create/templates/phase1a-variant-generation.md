以下の手順でベースライン作成とバリアント生成を行ってください:

1. Read で以下のファイルを読み込む:
   - {proven_techniques_path} （実証済みテクニック — ベースライン構築ガイドを含む）
   - {approach_catalog_path} （アプローチカタログ — 共通ルール・推奨構成・改善戦略を含む）
   - {reference_agent_path} （エージェント定義のひな形 — 構造の参考用）
2. エージェント定義ファイル {agent_path} を Read で確認する:
   - 存在すれば、その内容をベースライン（比較基準）とする
   - 存在しなければ（「新規」と指定された場合）、以下の手順で新規作成する:
     a. {user_requirements} を基に、proven-techniques.md の「ベースライン構築ガイド」に従って生成する
     b. reference_agent_path の構造（YAML frontmatter → ロール → 優先順位 → 基準 → 姿勢 → 出力）を参考にする
     c. 指示文は英語で記述し、テーブル/マトリクス構造を活用する
3. ベースラインを {prompts_dir}/v001-baseline.md として Write で保存する
4. エージェント定義ファイルが存在しなかった場合:
   - ベースラインの内容（Benchmark Metadata コメントを除く）を {agent_path} に Write で保存する（初期デプロイ）
5. ベースラインを以下の6つの構造次元で分析する:
   - 見出し数、サブ項目粒度、出力形式詳細度、原則/制約の明示度、具体例の有無、スコアリング基準の有無
6. 構造分析のギャップに基づき、approach-catalog.md からギャップが大きい次元の2つの独立変数を選定する
   - proven-techniques.md の効果データを参照し、実証済み高効果テクニック（✓マーク付き）を優先的に選択する
   - 回避すべきアンチパターンに該当するテクニックは選択しない
7. 各バリアントを {prompts_dir}/v001-variant-{name}.md として保存する
   - 各ファイル先頭に Benchmark Metadata コメントを記載する（Variation ID を必ず含める）
8. 以下のフォーマットで結果サマリのみ返答する（プロンプト本文は含めない）:

## エージェント定義
- エージェント名: {agent_name}
- 目的: {1行サマリ}
