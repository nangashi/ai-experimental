以下の手順でテストシナリオセットとルーブリックを生成してください:

1. Read で以下のファイルを読み込む:
   - {test_scenario_guide_path} （テストシナリオ作成ガイド）
   - {baseline_prompt_path} （ベースラインエージェント定義 — エージェントのタスク内容を理解するため）
   - {knowledge_path} （知見 — テスト履歴を確認し、シナリオの重複を避ける）
2. ベースラインのエージェント定義を分析し、以下を特定する:
   - エージェントの主要タスク（何をするエージェントか）
   - 想定入力（何を受け取るか）
   - 期待出力（何を生成するか）
   - 主要な能力カテゴリ（3-5個）
3. test-scenario-guide.md のガイドラインに従い、5-8個のテストシナリオを生成する:
   - 各シナリオにはシナリオID（T01-T08）、難易度、カテゴリ、入力コンテンツ、品質ルーブリック、期待行動、アンチパターンを含める
   - 入力コンテンツは20-100行で、実在のプロジェクト/タスクとして自然な内容にする
   - ルーブリックの各基準には Full(2)/Partial(1)/Miss(0) の具体的な条件と Weight を記載する
   - 難易度の分布: 易 2個 / 中 3個 / 難 1-3個
   - 全能力カテゴリをカバーする
4. テストシナリオを {test_set_save_path} に Write で保存する
5. ルーブリック（全シナリオの採点基準をまとめたもの）を {rubric_save_path} に Write で保存する
   - ルーブリックファイルにはシナリオごとの基準テーブル、Expected Key Behaviors、Anti-patterns を含める
6. 以下のフォーマットでサマリのみ返答する:

## テストシナリオセット
- エージェント: {agent_name}
- シナリオ数: {N}
- 能力カテゴリ: {カテゴリリスト}

| ID | タイトル | 難易度 | カテゴリ | 基準数 |
|----|---------|--------|---------|--------|
| T01 | {title} | {difficulty} | {category} | {N} |
| T02 | {title} | {difficulty} | {category} | {N} |
...
