# Output Template

Phase 3 でタスクリスト文書を生成する際に参照するテンプレート。

## テンプレート

```markdown
# タスクリスト: {プロジェクト名}

## ステータス

Draft

## 日付

{YYYY-MM-DD}

## 概要

- タスク数: {N}
- 入力文書:
  - `{ps_path}` ({日付})
  - `{req_path}` ({日付})
  - `{arch_path}` ({日付})
  - `{std_path}` ({日付})
  - `{dp_path}` ({日付})

## 依存関係グラフ

{mermaid 図}

## カバレッジサマリ

| 要件種別 | カバー | 合計 |
|---------|-------|------|
| SR | {M} | {N} |
| NFR | {M} | {N} |
| IR | {M} | {N} |

## タスク一覧

### [{1}/{N}] {タイトル}

**ブロック**: {blocked_by があれば「Task {X}（{タイトル}）」、なければ「なし」}

## 背景

{problem_summary}を解決するため、{approach}として開発する。
本タスクは全{N}タスク中の第{M}タスクであり、{このタスクの位置づけ・目的を1-2文}。

前提: {blocked_by があれば「Task {X}（{タイトル}）が完了していること」、なければ「なし」}

## やること

- {具体的な実装ステップ}
- {...}

## 受け入れ基準

- [ ] {検証可能な条件}
- [ ] {...}

## 制約

- {このタスクの実装に影響する制約}
- {...}

## スコープ外

- {他タスクで扱う機能}
- {...}

---

### [{2}/{N}] {タイトル}
...
```

## 記入ガイダンス

- **ブロック**: `blocked_by` の依存関係をタスク番号とタイトルで記述する
- **タスク区切り**: 各タスク間は `---` で区切る
- **mermaid 図**: 依存関係グラフは mermaid の graph TD 形式で記述する
- **カバレッジ**: QG-1 で検証済みのカバレッジ数値を転記する
- **イシュー本文**: Phase 2 で生成済みの `task-bodies.md` の各タスク本文をそのまま転記する
