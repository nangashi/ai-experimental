# Output Template

Phase 3 でタスクリスト文書を生成する際に参照するテンプレート。マスターファイルと個別タスクファイルの2種類。

## マスターファイルテンプレート

`{output_path}`（`docs/project-definition/task-list.md`）に出力する。

```markdown
# タスクリスト: {プロジェクト名}

## ステータス

Draft

## 日付

{YYYY-MM-DD}

## 概要

- タスク数: {N}
- 入力文書:
  - `{ps_path}` ({日付})
  - `{req_path}` ({日付})
  - `{arch_path}` ({日付})
  - `{std_path}` ({日付})
  - `{dp_path}` ({日付})
  - `{dd_path}` ({日付})（存在する場合）

## 依存関係グラフ

{mermaid 図}

## カバレッジサマリ

| 要件種別 | カバー | 合計 |
|---------|-------|------|
| SR | {M} | {N} |
| NFR | {M} | {N} |
| IR | {M} | {N} |

## タスク一覧

| # | ファイル | タイトル | 実行者 | 元スライス | blocked_by |
|---|---------|---------|--------|-----------|-----------|
| 001 | [001-{slug}.md](task-list/001-{slug}.md) | {タイトル} | {llm/human} | ルール{N} — {スライス名} | — |
| 002 | [002-{slug}.md](task-list/002-{slug}.md) | {タイトル} | {llm/human} | ルール{N} — {スライス名} | 001 |
| ... | | | | | |
```

## 個別タスクファイルテンプレート

`{output_dir}/{NNN}-{slug}.md` に出力する。

```markdown
# [{NNN}/{total}] {タイトル}

**ブロック**: {Task NNN（タイトル）— 利用する成果物の概要 / なし}

## 目的

全{total}タスク中の第{NNN}タスク。{このタスクが何を実現するか（1-2文）}。

## 実行者

{`llm` — LLMがコード実装を行う / `human` — 人間が外部サービスのWebコンソール上で作業を行い、LLMは受け入れ基準の検証のみ行う}

## 受け入れ基準

- [ ] {SR/NFR から導出した検証可能な条件}

## 入力

- `detailed-design.md` §{X}（{何の仕様か}）
- `architecture.md` §{X}（{何の仕様か}）
- `standards.md`（{関連する規約名}）
- `development-process.md`（{テスト戦略等}）
```

## 記入ガイダンス

- **ブロック**: 依存タスクの番号・タイトルと、利用する成果物の概要を記述する
- **タスク区切り**: 個別ファイルのため区切りは不要
- **mermaid 図**: 依存関係グラフは mermaid の graph TD 形式で記述する
- **カバレッジ**: QG-1 で検証済みのカバレッジ数値を転記する
- **イシュー本文**: Phase 2 で生成済みの `task-bodies.md` の各タスク本文を個別ファイルに転記する
- **slug 生成ルール**: タスクの主要な内容を英語で短縮した名前を使用する（例: `toolchain-setup`, `auth-config`, `core-data-repository`）。小文字ハイフン区切り、20文字以下を目安とする
