以下の手順で採点とエラー分析を実行してください:

## 採点

1. Read で以下を読み込む:
   - {scoring_rubric_path} （採点基準）
   - {answer_key_path} （正解キー — この文書用）
   - {perspective_path} （観点定義）
   - {result_run1_path} （Run1結果）
   - {result_run2_path} （Run2結果）

2. 採点基準の検出判定基準に従い、各 Run の結果を問題ごとに ○/△/× で判定する。以下のアンカーを参照し判定の一貫性を確保する:

   **○判定**: 正解キーの検出判定基準を**すべて**満たす指摘がある。核心的な問題点と具体的な対策の両方に言及。
   **△判定**: 関連カテゴリへの言及はあるが、核心的な問題点に到達していない。一般的・抽象的な言及にとどまる。
   **×判定**: 当該問題に関する指摘が全くない。

3. 各 Run のスコアを計算する:
   ```
   run_score = Σ(検出スコア) + (ボーナス件数 × 0.5) - (ペナルティ件数 × 0.5)
   ```

4. カテゴリ別検出率を計算する:
   ```
   category_rate = (Σ○ + 0.5 × Σ△) / (category_questions × 2)
   ```

## エラー分析（抽象的）

5. **重要: 問題の具体的なタイトル(P01等)や文書固有の内容は含めないこと。カテゴリレベルの抽象的な分析のみ行う。**

   a. **カテゴリ別検出状況**: カテゴリごとに○/△/×数と安定性を集計する
   b. **プロンプトギャップ**: 未検出/部分検出カテゴリについて、プロンプトのどの要素が不足しているかを分析する
   c. **改善方向**: 各ギャップに対する改善の方向性（方向性のみ、具体的修正案は含めない）
   d. **保持すべき強み**: 完全検出カテゴリについて、有効なプロンプト要素を分析する
