以下の手順で観点定義の抽出、初期プロンプト、バリアントを生成してください:

1. Read で以下のファイルを読み込む:
   - {perspective_template_path} （観点テンプレート）
   - {approach_catalog_path} （アプローチカタログ — 共通ルール・推奨構成・改善戦略を含む）
2. Phase 0 で perspective.md は既にコピー済みのため、{perspective_path} が存在することを Read で確認する
3. エージェント定義ファイル {agent_definition_path} を Read で確認する:
   - 存在すれば、その内容をベースライン（比較基準）とする
   - 存在しなければ、アプローチカタログの推奨構成と .claude/agents/ 配下の類似エージェントを参考に新規作成する
4. ベースラインを prompts/v001-baseline.md として Write で保存する
5. エージェント定義ファイルが存在しなかった場合:
   - ベースラインの内容（Benchmark Metadata コメントを除く）を {agent_definition_path} に Write で保存する（初期デプロイ）
6. ベースラインを以下の6つの構造次元で分析する:
   - 見出し数、サブ項目粒度、出力形式詳細度、原則/制約の明示度、具体例の有無、スコアリング基準の有無
7. 構造分析のギャップに基づき、approach-catalog.md からギャップが大きい次元の2つの独立変数を選定し、各バリアントを生成する:
   - 各バリアントを prompts/v001-variant-{name}.md として保存する
   - 各ファイル先頭に Benchmark Metadata コメントを記載する（Variation ID を必ず含める）
8. 以下のフォーマットで結果サマリのみ返答する（プロンプト本文は含めない）:

## 観点定義
- 観点: {perspective}
- 対象: {target}
- 評価スコープ: {主要項目をカンマ区切りで列挙}

## 構造分析結果
| 構造次元 | 現状 | 最適状態 | ギャップ |
|---------|------|---------|---------|
...
