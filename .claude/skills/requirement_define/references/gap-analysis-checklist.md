# Information Gap Analysis Checklist

Phase 1 で入力文書の情報充足度を評価するためのチェックリスト。
チェック範囲はこの7カテゴリに限定する（AIが独自にカテゴリを追加しないこと）。

## チェック項目

### A. 認証・アクセスモデル

- 認証方式の必要性（ログイン有無、シングルユーザー/マルチユーザー）
- アクセス制御の範囲（公開/非公開、ネットワーク保護のみ等）

影響先: Phase 2（システム境界）、Section 4（制約）

### B. データ入力方法

- 各デバイスからのデータ入力方法（手入力、共有インテント、クリップボード、ブックマークレット等）
- 入力フローの前提（どのアプリから、どのような操作で遷移するか）

影響先: Phase 3（SR導出）、Section 7（システム境界）

### C. データ量・ライフサイクル

- 想定データ量（オーダー: 数十件 / 数百件 / 数千件以上）
- データ保持期間（永続 / 自動アーカイブ / TTL）
- データ増加ペースの想定

影響先: Phase 4（NFR定量化）、Section 9（非機能要件）

### D. 接続性の前提

- オフライン操作の必要性（保存・閲覧がオフラインで可能か）
- データ同期のタイミング（リアルタイム / 次回アクセス時 / 手動）

影響先: Phase 2（システム境界）、Section 9（非機能要件）

### E. 曖昧な用語

- 定義なしで使用されている用語（複数解釈が可能なもの）
- 測定不能な表現（「素早く」「一般的に」「最低限」等）

影響先: Phase 3（SR導出時の解釈）

### F. NFR測定精度

- 定性的のみで数値の手がかりが全くないNFR項目
- 測定対象の境界が不明確なNFR項目（例: 応答時間に外部API呼び出しを含むか）

影響先: Phase 4（NFR定量化の入力）

### G. 統合境界

- 外部サービス連携の詳細（どのサービス/API、認証方式、レート制限）
- フォールバック動作の前提（外部サービス障害時の振る舞い）

影響先: Phase 5（インターフェース要件）

## 分類基準

各チェック項目を以下に分類する:

| 分類 | 基準 | 対応 |
|------|------|------|
| Resolved | 入力文書に明示的な記述がある | 該当箇所を引用して記録。以降のPhaseで参照 |
| Inferable | 明示されていないが文脈から合理的に推定可能 | 推定内容と根拠を記録し、Step 3 でユーザーに確認 |
| Gap | 記載なし、推定も困難 | Step 2 でユーザーに質問 |
