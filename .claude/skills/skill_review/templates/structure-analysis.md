# 構造分析

スキル定義のワークフロー構造を可視化し、コンテキスト消費量を定量化し、参照整合性とスキル外参照をチェックする。

## 入力

- `{skill_path}`: 対象スキルディレクトリの絶対パス
- `{skill_name}`: スキル名
- `{file_list_path}`: ファイルリストのパス
- `{report_save_path}`: レポート保存先パス

## 手順

### Step 1: ファイル収集

1. `{file_list_path}` を Read してスキル内の全 .md ファイルパスを取得する
2. 全ファイルを Read する

### Step 2: フロー図の抽出

SKILL.md のマークダウン構造から Phase → Step → Task の階層を抽出し、以下の形式でフロー図を構成する:

```
Phase 0: {Phase名}
  └─ {処理概要}

Phase 1: {Phase名} (並列)
  ├─ [{model}] {Task名/テンプレート名} → {出力ファイル}
  └─ [{model}] {Task名/テンプレート名} → {出力ファイル}

Phase 2: {Phase名} (対話)
  ├─ Step 1: {処理概要}
  └─ Step 2: {処理概要}
    └─ 分岐: {条件} → {遷移先}
```

抽出ルール:
- Task 呼び出しには `[model]` を付記する（model 指定がない場合は `[default]`）
- ファイルへの Write/保存がある場合は `→ {ファイル名}` を付記する
- 条件分岐がある場合は分岐先を含める
- AskUserQuestion がある Phase には `(対話)` を付記する

### Step 3: コンテキスト消費マップ

以下を集計してテーブル形式で構成する。

**親コンテキスト（常駐）:**

| ファイル | 行数 |
|---------|------|
| SKILL.md | {N} |

**サブエージェント別コンテキスト:**

各 Task 呼び出しについて、読み込むファイルと行数を集計する:

| サブエージェント | 読み込むファイル | 行数 | タイミング |
|---------------|---------------|------|----------|
| {Task名} | {テンプレート名} | {N} | Phase {N} |

**親に戻る情報量:**

各 Task の返答サイズを推定する:
- 「〜とだけ返答する」等の最小化指示がある → 「最小（〜5行）」
- ファイルに保存して返答する → 「最小（保存確認のみ）」
- 返答最小化指示なし → 「制約なし（要注意）」

### Step 4: 参照チェック

#### A1: 存在しないファイル参照

SKILL.md と全テンプレート内からファイルパスを抽出する。対象パターン:
- バッククォート内のパス
- テンプレート参照（`templates/name.md`, `references/name.md`）

以下は除外する:
- `{variable}` のみで構成されるパス（ランタイム生成。例: `{work_dir}/output.md`）
- Write/保存の出力先パス
- URL（`http://`, `https://`）

除外後の各パスについて:
- 相対パスの場合は `{skill_path}` を基準に絶対パスに解決する
- Glob または Read で存在を確認する
- 存在しないパスを記録する

#### B3: パス変数名の不一致

1. SKILL.md 内の各 Task 呼び出しからパス変数定義（`- \`{variable}\`:` 形式）を抽出する
2. 呼び出し先テンプレートの「入力」セクションから変数定義を抽出する
3. 片方にあって他方にない変数を検出する
4. 類似名ペア（部分文字列、1-2文字の差異）を特定する

#### EXT: スキル外参照

SKILL.md と全テンプレート内から、以下に該当するパスを収集する:
- `.claude/skills/` を含むパスで、`.claude/skills/{skill_name}/` 以外を指すもの
- `.claude/instructions/` を含むパス
- プロジェクト内の他ディレクトリへの明示的な参照

各参照について、参照元ファイル・該当箇所・参照先パスを記録する。

### Step 5: レポート出力

以下の構造で `{report_save_path}` に Write する:

```
# 構造分析レポート: {skill_name}

## フロー図

{Step 2 の出力}

## コンテキスト消費マップ

{Step 3 の出力}

## 参照チェック

### A1: 存在しないファイル参照
{検出数}件
{各件の詳細リスト、0件なら「なし」}

### B3: パス変数名の不一致
{検出数}件
{各件の詳細リスト、0件なら「なし」}

## スキル外参照
{EXT の結果。参照元 → 参照先 の形式で列挙。0件なら「なし」}
```

最後に「保存完了: {report_save_path}」とだけ返答する。
