# 設計レビュー

スキル定義のワークフロー設計を、確立された設計原則に基づいてレビューし、改善すべきパターンを検出する。

## 入力

- `{skill_path}`: 対象スキルディレクトリの絶対パス
- `{skill_name}`: スキル名
- `{skill_dir}`: skill_review スキルのディレクトリの絶対パス
- `{file_list_path}`: ファイルリストのパス
- `{findings_save_path}`: 結果保存先パス

## 手順

### Step 1: ファイル収集

1. `{file_list_path}` を Read してスキル内の全 .md ファイルパスを取得する
2. 全ファイルを Read する
3. `{skill_dir}/references/review-patterns.md` を Read してレビューパターン定義を取得する

### Step 2: パターン検出

review-patterns.md に定義された各パターンについて、以下の手順で検出を行う:

1. パターンの「検出条件」を SKILL.md とテンプレートの内容に照合する
2. 該当箇所が見つかった場合、以下を記録する:
   - パターン ID とパターン名
   - 重要度（パターン定義のデフォルト重要度に従う。対象スキルの文脈で影響が明確に異なる場合は調整し、理由を付記する）
   - 該当箇所（ファイル名:セクション名 or 行範囲）
   - 具体的にどの記述が問題パターンに該当するかの説明
   - 改善案（パターン定義の改善方針を対象スキルの文脈に具体化したもの）
   - 根拠（参照する instruction ファイル名と原則名）

**検出の原則:**
- 該当箇所を具体的に引用すること。「〜の可能性がある」で終わる検出は、発生条件を具体的に示せない場合は記録しない
- パターン定義にない問題を追加で検出しない。定義されたパターンのみをチェックする
- コマンドがエラーを返した場合、またはデータが空・期待する構造と異なる場合は、以降のステップを続行せず、エラー内容と影響範囲を返却に含めて即座に返却する

### Step 3: 結果出力

検出された全 finding を重要度順（高 → 中 → 低）で `{findings_save_path}` に Write する:

```
# 設計レビュー結果: {skill_name}

検出: {N}件 (高: {N}, 中: {N}, 低: {N})

### {ID}: {パターン名}
- **重要度**: {高/中/低}
- **場所**: {ファイル名}:{セクション名 or 行範囲}
- **問題**: {具体的な問題の説明。該当箇所の引用を含む}
- **改善案**: {対象スキルの文脈に即した具体的改善提案}
- **根拠**: {instruction ファイル名}「{該当原則の引用}」
```

0件の場合:
```
# 設計レビュー結果: {skill_name}

検出: 0件
```

最後に「保存完了: {findings_save_path}, 検出: {N}件 (高: {N}, 中: {N}, 低: {N})」とだけ返答する。
