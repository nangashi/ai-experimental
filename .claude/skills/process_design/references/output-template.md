設計原則:
- commit 時: 高速なチェックのみ（変更ファイル対象の lint/format）
- push 時: 中量のチェック（型チェック等）
- CI: 全量チェック（テスト全量、ビルド検証、セキュリティスキャン）

## 記入ガイダンス

- **P1/P2 導出根拠**: 各セクションで、プロセスが P1/P2 のどの決定から導出されたかを明示する
- **品質チェック配置**: P2 の自動強制サマリ（保存時/commit 時のチェック）と P3 の品質チェック配置サマリに矛盾・重複がないことを確認する
- **具体性**: 「適切に設定する」等の曖昧な記述を避け、具体的なツール名・設定方針を記述する
- **代替案の排除**: 「AまたはB」のような未確定の選択肢を残さない
- **ADR参照**: ADRが存在する選定はADRファイルへの参照を含める
- **該当なし**: 該当しないセクションは省略せず「該当なし」と明記する
- **AI 統合**: AI 統合プロセスが省略された場合はセクション 5.5〜5.7 を「省略（AI 統合は MVP 後に決定）」と記載する
