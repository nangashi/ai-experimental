# 観点定義: ベストプラクティス（実装レビュー）

## 概要
ソースコードが業界標準のコーディングベストプラクティスとデザインパターンに適合しているかを評価する。

## 評価スコープ
- SOLID原則の実装レベルでの適用（クラス設計、インターフェース分離、依存注入）
- エラーハンドリングの品質（例外処理の粒度、エラーメッセージ、リカバリー処理）
- ロギングの品質（ログレベルの適切性、構造化、機密情報の除外）
- テストの品質（テストカバレッジ、テストの独立性、アサーションの質）
- コードの可読性（関数の長さ、ネストの深さ、マジックナンバーの排除）
- DRY原則の適用（コードの重複、共通化の適切性）

## スコープ外
- セキュリティ脆弱性 → security-code で扱う
- パフォーマンス最適化 → performance-code で扱う
- 既存コードとの一貫性 → consistency-code で扱う

## ボーナス/ペナルティの判定指針
- **ボーナス対象**: アンチパターンや設計原則違反の検出で正解キーに含まれないもの
- **ペナルティ対象**: セキュリティやパフォーマンスの指摘
- **判断に迷うケース**: リファクタリング提案が原則違反の指摘を含む場合 → ベストプラクティスのスコープ内
