以下の手順で knowledge.md を更新してください:

1. Read で以下を読み込む:
   - {knowledge_path} （現在のナレッジ）
   - {report_save_path} （比較レポート）
2. knowledge.md を以下のルールで更新して保存する:
   - 推奨: {recommended_name}（判定根拠: {judgment_reason}）
   - 累計ラウンド数を +1
   - 初期スコアが未測定の場合、今回のベースラインスコアを初期スコアとして記録する
   - 効果的だったバリアントの変化 → 「効果が確認された構造変化」テーブルに追記
   - 同等以下だったバリアントの変化 → 「効果が限定的/逆効果」テーブルに追記
   - 「バリエーションステータス」テーブルで、今回検証したバリエーション ID の Status を EFFECTIVE/INEFFECTIVE/MARGINAL に更新し、Round と Effect を記入する
   - 「テストセット履歴」に今回のラウンドのエントリを追記
   - 「ラウンド別スコア推移」テーブルに今回のラウンドのエントリ（各プロンプトの Mean(SD)、Best スコア、Δ from Initial）を追記する
   - 「最新ラウンドサマリ」を今回のラウンドの結果で上書きする（最大10行: ラウンド番号、スコア一覧、推奨、主要知見）
   - 「改善のための考慮事項」を以下のルールで更新する:
     - 既存の原則を全て保持する（削除しない）
     - 新ラウンドで得られた知見を既存原則に統合する（矛盾する場合は新しい知見で更新し、旧原則を修正する）
     - 新たに一般化可能な原則があれば追加する
     - 各原則は「[原則] （根拠: Round N, バリアントX, 効果±Xpt）」の形式とする
     - 20行を超える場合は、最も根拠が弱い（効果ptが小さい/SDが大きい）原則を統合または削除する
     - 個別ラウンドの詳細分析は記載しない（詳細はレポートファイルに存在する）
3. 以下のフォーマットで確認のみ返答する:

knowledge.md 更新完了（累計ラウンド数: {N}）
