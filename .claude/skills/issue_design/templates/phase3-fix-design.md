以下の手順でレビュー指摘に基づいて設計ドキュメントを修正してください:

## 手順

### 1. 要件の把握

{requirements_path} を Read で読み込み、設計が満たすべき要件を把握する。

### 2. 現在の設計の把握

{design_path} を Read で読み込み、現在の設計内容を把握する。

### 3. レビュー指摘の把握

{consolidated_path} を Read で読み込み、Critical と Significant の指摘を把握する。

### 4. 設計修正

Critical と Significant の各指摘に対して設計を修正する:

- **推奨事項を設計に反映する**: 指摘の「推奨」欄の内容を設計に取り込む
- **要件との整合性を維持する**: 修正が要件に矛盾しないことを確認する
- **設計箇所を特定して修正する**: 指摘の「設計箇所」欄を参照し、該当セクションを修正する
- **修正不可能な指摘**: 要件上の制約等で対応できない場合は、理由を記録する

修正方針:
- Critical は必ず対応する（対応不可の場合は明確な理由が必要）
- Significant は可能な限り対応する
- Moderate / Minor は対応しない（次のレビューサイクルまたは実装時に対応）

### 5. 修正後の設計を保存

修正後の設計全体を {design_path} に Write で上書き保存する。

### 6. 返答

以下のフォーマットで返答する:

```
addressed: {対応した指摘数}
skipped: {対応不可の指摘数}
skipped_reasons: {対応不可の理由リスト（あれば。なければ「なし」）}
summary: {修正内容の1行サマリ}
```
