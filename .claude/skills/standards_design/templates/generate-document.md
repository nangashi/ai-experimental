以下の手順で開発規約書を生成してください:

## 手順

### 1. テンプレートの読み込み

{template_path} を Read で読み込み、出力フォーマットと記入ガイダンスを把握する。

### 2. 入力の把握

{arch_path} を Read で読み込み、以下を把握する:
- 技術スタック
- コンポーネント構成
- 横断的関心事（エラーハンドリング方針、ログ・監視、P2 への引き継ぎ事項）
- 認証方式・セッション管理

{req_path} を Read で読み込み、以下を把握する:
- 非機能要件（NFR一覧）
- 制約

### 3. 設計決定事項の把握

{decisions_path} を Read で読み込み、Phase 1 で確定した全情報を把握する:
- Linter/Formatter 選定結果（ADR 参照含む）
- グループ別規約一覧（各項目の確定規約 + 根拠 + 自動強制/手動遵守の区分）
- N/A 項目一覧と理由
- P1 引き継ぎ事項の反映状況

### 4. ADRの参照

{adr_dir} を Glob で確認し、存在するADRファイルの「決定」セクションを Read で把握する。

### 5. 開発規約書の生成

テンプレートに従い、設計決定事項・入力情報・ADRを統合して開発規約書を生成する。

生成ルール:
- 各セクションの内容は {decisions_path} の確定情報に忠実に従う
- ADRが存在する選定はADRファイルへの参照を含める（例: `ADR-0001`）
- 「該当なし」のセクションは省略せず「該当なし」と明記する
- 曖昧な記述（「適切に設定する」等）を避け、具体的な設定・パターンを記述する
- 「AまたはB」のような未確定の選択肢を残さない
- P1 導出根拠: 各規約セクションで、その規約が P1 のどの決定から導出されたかを明示する
- 自動強制サマリ: 各規約について、選定ツール（Linter/Formatter）で自動強制可能か、手動遵守のみかをセクション 8 で明確にする
- P3 引き継ぎ: P3（開発プロセス設計）に引き継ぐ横断的関心事をセクション 9 に記載する
- P1 トレーサビリティ: P1 決定と導出された P2 規約の対応をセクション 10 に記載する
- AI 統合規約が省略された場合はセクション 7 ごと「省略（AI 統合は MVP 後に決定）」と記載する

### 6. 保存

生成した開発規約書を {output_save_path} に Write で保存する。ステータスは `Draft` とする。

### 7. 返答

以下のフォーマットで返答する:

```
result: success
sections_completed: {完了セクション数}
p1_traceability: {P1トレーサビリティテーブルの行数}
auto_enforcement: {自動強制可能な規約数}/{全規約数}
```
