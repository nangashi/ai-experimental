# AIコーディング・アンチパターン

AIコーディングエージェントに固有の品質劣化パターン。コード生成・編集時に自己チェックとして使用する。

## デッドコード蓄積 (Dead Code Accumulation)

新しい実装を追加する際に、置き換えた旧実装を削除しない傾向。

**発生パターン**:
- 関数/メソッドのリファクタ時に旧版を残したまま新版を追加する
- 条件分岐の変更時に到達不能なブランチを残す
- import文やuse宣言が不要になっても削除しない

**自己チェック**: 変更完了後に「この変更で不要になったコード（旧実装、未使用import、到達不能ブランチ）はないか」を確認する。

## 隣接コード変更 (Adjacent Code Alteration)

タスクの対象コードの近くにあるという理由だけで、無関係なコードを変更する傾向。

**発生パターン**:
- 対象関数の修正時に、同じファイル内の別の関数のスタイルを「ついでに」修正する
- コメントの追加・削除・修正を副作用的に行う
- 変数名のリネームやフォーマット修正を要求外で行う

**自己チェック**: 変更差分の各箇所について「この変更はタスクの要件に直接必要か」を確認する。必要でない変更は含めない。
