# improve_seq スキル — コンセプト

## 目的

AIレビューの表面性・部分最適を克服し、対象の構造を深く理解した上で有効な改善のみを適用する。「AIによる改善がどこまで有効か」を把握するための実験的スキル。

## 背景にある課題

- AIレビューが表面的で、部分最適な修正を提案する
- 過剰な修正提案があり、都度判断する手間がかかる
- 全体の構造・目的を理解せず局所的に「良く」しようとする

## 核心のアプローチ: 理解してからレビューする

対象を直接レビューするのではなく、まず対象の構造・目的・制約を深く分析し、その理解を基盤としてレビューする。これにより:
- レビュアーが全体像を把握した上で指摘するため、部分最適を回避できる
- 「この対象にとって何が重要か」が明示されるため、些末な指摘が減る
- 改善適用時も構造理解を参照することで、全体の一貫性を維持できる

## インターフェース

```
/improve_seq ${key} ${target_path}
```

- `key`: ワーキングディレクトリのキー（セッション識別子）
- `target_path`: 改善対象ファイルのパス

## 出力先

```
.skill_output/improve_seq/${key}/
```

## 設計原則

### P1: 構造理解の先行

レビューの前に、対象のフロー・目的・制約・設計意図を言語化する。この構造理解を全後続フェーズの文脈として渡す。

### P2: 有効性のある改善の識別

各指摘に対し「対象が複雑化しないか」「修正で本当に効果が見込めるか」を構造的にチェックし、些末な指摘やLLM特有の過剰最適化を排除する。

### P3: 最小限の人間判断

トレードオフのない改善は自動適用し、判断が必要なもののみ人間に提示する。人間の判断結果を記録し、同一・類似の指摘を繰り返さない。

## 生成と評価の分離

- **構造理解（分析）**: 対象の構造を分析するサブエージェント
- **レビュー（評価）**: 構造理解を基盤として対象を批判的に分析するサブエージェント
- **改善適用（生成）**: 構造理解 + レビュー結果を受けて修正を適用するサブエージェント
- **トリアージ（判定）**: 指摘の有効性を判定する親エージェント

各役割を独立したコンテキストで実行することで、自己選好バイアスを構造的に防止する。
