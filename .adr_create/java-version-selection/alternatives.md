# 代替案一覧

## 採用候補

### ALT-1: Java 17（LTS）
- 概要: 2021年リリースのLTS。エコシステムが最も成熟しており、Spring Boot 3の最小要件。安定性と豊富な情報を重視する保守的な選択肢。
- 提案元: OBJ-1（長期的な保守性）、OBJ-2（エコシステム成熟度）、OBJ-3（学習コスト）、OBJ-5（将来の柔軟性）

### ALT-2: Java 21（LTS）
- 概要: 2023年リリースのLTS。Virtual Threads、Generational ZGC等の革新的機能を持ち、性能と将来性のバランスに優れる選択肢。
- 提案元: OBJ-1（長期的な保守性）、OBJ-4（パフォーマンス）、OBJ-5（将来の柔軟性）

### ALT-3: Java 25（LTS）
- 概要: 2025年9月リリースの最新LTS。最長のサポート期間と最新機能を持つが、2026年2月時点ではリリース後5ヶ月でエコシステムが未成熟。
- 提案元: OBJ-4（パフォーマンス）、OBJ-5（将来の柔軟性）

## 除外した代替案
- Java 8: エコシステムから完全離脱済み、Spring Boot 3.x利用不可、新規プロジェクトに不適切 — C3, C1
- Java 11: Oracle JDK EOLが2026年9月に迫り、Spring Boot 3.x利用不可（C3）。S4/S10の仮定（長期運用）と矛盾 — C3, S7
- Java 23（非LTS）: Premier Supportが6ヶ月のみで長期運用に不適。S4/S10の仮定と矛盾 — S4, S10
