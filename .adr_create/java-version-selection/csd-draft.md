# CSD分類

## 確実（Certainty）

- C1: プロジェクトは新規開発である — 根拠: 決定ステートメントで「新規開発プロジェクト」と明示されており、既存プロジェクトのマイグレーションは対象外と定義されている
- C2: Java 11のOracle JDKサポートは2026年9月にEOLを迎える — 根拠: Oracle公式サポートロードマップで確定済み。新規開発での採用は推奨されない
- C3: Spring Boot 3.xはJava 17を最小要件とする — 根拠: Spring Boot公式リリースノートで明示。Java 8/11では利用不可
- C4: Java LTSバージョンは17, 21, 25が現時点で選択可能 — 根拠: Oracleのリリーススケジュールで確定。次回LTSは2027年9月のJava 29
- C5: Javaの後方互換性は比較的高い — 根拠: 決定ステートメントで可逆性がmediumと評価されており、段階的な移行が技術的に可能
- C6: JDKディストリビューション（Oracle JDK vs OpenJDK系）の選択は対象外 — 根拠: 決定ステートメントのスコープで明示的に除外されている

## 仮定（Supposition）

- S1: プロジェクトで主要なJavaフレームワーク（Spring Boot等）を使用する — 根拠: 多くのエンタープライズJavaプロジェクトで標準的。ただしフレームワークレスの選択肢もある、検証方法: プロジェクトの技術スタック要件を確認
- S2: Java 21のVirtual Threadsが並行処理で有効に機能する — 根拠: ベンチマーク結果で高並行性アプリケーションでの性能向上が報告されている。ただしアプリケーション特性に依存する、検証方法: プロジェクトの並行処理要件とアーキテクチャを評価
- S3: OpenJDKディストリビューション（Eclipse Temurin等）が無償で利用可能な状態が継続する — 根拠: 現時点で複数のベンダーが無償提供中。ただし将来のビジネスモデル変更の可能性はある、検証方法: 主要ディストリビューターのライセンスポリシーを定期的に確認
- S4: プロジェクトの開発期間は2年以上継続する — 根拠: 一般的なエンタープライズ開発の標準的な期間。LTSサポート期間が重要になる、検証方法: プロジェクト計画書で開発・運用期間を確認
- S5: チームメンバーはJava 8/11の経験はあるが、Java 17以降の新機能には習熟していない — 根拠: 多くの企業でJava 8/11が長期利用されてきた一般的な状況。ただしチーム構成に依存する、検証方法: チームスキルセットの調査を実施
- S6: パフォーマンスが重要な要件である — 根拠: リサーチでJava 17→21の性能差が調査されており、性能が考慮事項と推測。ただし実際の優先度は不明、検証方法: 非機能要件の優先順位を確認

## 不確実（Doubt）

- D1: プロジェクトで使用する予定のサードパーティライブラリのJava 21/25互換性 — 影響範囲: 特定のバージョンを選択した場合に、必要なライブラリが利用できない可能性がある、リスク: プロジェクト開始後にライブラリ互換性問題が発覚し、バージョン変更や代替ライブラリ検討が必要になる
- D2: チームの学習コスト許容度と教育リソースの確保 — 影響範囲: 新バージョンの機能習得に必要な時間と教育投資が不明、リスク: 学習コストが見積もりより大きく、開発スピードに影響する可能性
- D3: Oracle JDK vs OpenJDK系ディストリビューションの選択方針 — 影響範囲: 決定ステートメントでは対象外とされているが、Java 21のNFTC版期限（2026年9月）の影響を受ける可能性、リスク: ディストリビューション選択が未確定の場合、バージョン選択の評価に影響する
- D4: プロジェクトの並行処理要件の具体的な規模 — 影響範囲: Virtual Threads（Java 21の主要機能）の価値評価に影響、リスク: 並行処理が少ない場合、Java 21の性能的メリットが限定的になる可能性
- D5: インフラ環境（クラウド/オンプレ、コンテナ化の有無）の詳細 — 影響範囲: Generational ZGC等のGC最適化の効果が環境依存、リスク: インフラ環境次第で性能特性が変化し、バージョン選択の優先順位が変わる可能性
- D6: Java 25のエコシステム成熟度（2026年2月時点） — 影響範囲: 最新LTSだがリリース後5ヶ月程度。ツール・ライブラリ・IDE・ドキュメントの成熟度が不明、リスク: 早期採用による情報不足やツールサポート不足で開発効率が低下する可能性
